# 《余烬商队》第 6.3 类：能力槽、事件信息与队伍被动系统 v0.2

> 本文为第 6 类“角色、技能、成长与招募系统”下的修订文档。  
> v0.2 用新口径替换 v0.1 的旧口径。  
> 旧版 v0.1 的核心是“每名角色 2 个固定被动词条”；新版 v0.2 改为“每名角色 4 个能力槽：2 个数值被动 + 2 个事件 / 世界接口”。

---

## 一、系统定位

第 6.3 类用于解决：

> 角色在战斗之外，如何影响野外事件、聚落事件、信息显示、路线判断、资源事件、车队损失、内部压力和世界互动。

它不是战斗技能系统。  
战斗技能成长放到后续 **6.4 初始角色技能成长系统** 中处理。

6.3 主要处理四件事：

| 模块 | 说明 |
|---|---|
| 能力槽结构 | 每名角色拥有 2 个数值被动 + 2 个事件 / 世界接口 |
| 信息等级 | 角色如何提高事件信息、路线信息、风险提示 |
| 权重修正 | 角色如何降低坏结果权重、提高好结果权重 |
| 事件接口 | 角色如何在事件中提供额外选项、替代选项、提示或特殊处理方式 |

一句话定位：

> **6.3 不是让角色多几个战斗技能，而是让角色在战斗外也有专业存在感。**

---

## 二、v0.2 相对 v0.1 的修正

| 项目 | v0.1 旧口径 | v0.2 新口径 |
|---|---|---|
| 每名角色能力数量 | 2 个固定被动词条 | 4 个能力槽 |
| 能力类型 | 全部是被动 | 2 个数值被动 + 2 个事件 / 世界接口 |
| 是否影响事件选项 | 基本不直接影响 | 可以影响事件提示、选项和结果 |
| 是否常驻主动技能 | 否 | 否 |
| 是否战斗技能 | 否 | 否 |
| 是否用于聚落事件 | 较弱 | 可以接入聚落事件 |
| 是否用于支线 / 灰兽潮 / 遗迹 | 较弱 | 后续可接入 |

新版目标：

> v0.1 解决“角色有后台被动”。  
> v0.2 进一步解决“角色在事件里有存在感”。

---

## 三、能力槽总结构

每名角色基础拥有 4 个能力槽：

| 能力槽类型 | 数量 | 作用 |
|---|---:|---|
| 数值被动 | 2 个 | 后台固定生效，影响权重、信息、修车、资源、损失等 |
| 事件 / 世界接口 | 2 个 | 在特定事件中出现额外处理方式、提示或特殊选项 |
| 合计 | 4 个 | 形成角色在战斗外的专业性 |

基础格式：

```text
角色
├─ 数值被动 1
├─ 数值被动 2
├─ 事件 / 世界接口 1
└─ 事件 / 世界接口 2
```

---

## 四、数值被动规则

### 4.1 数值被动是什么

数值被动是角色的后台专业能力。

它可以影响：

- 事件结果权重；
- 信息等级；
- 资源收益权重；
- 修车效率；
- 损失权重；
- 内部压力；
- 车队安全；
- 特定事件的风险分布。

例如：

| 角色 | 数值被动 | 作用 |
|---|---|---|
| 护路人 | 护卫 | 降低敌对、夜袭、混乱类事件最差结果权重 |
| 荒野射手 | 野外侦察 | 路线、敌情、痕迹信息等级 +1 |
| 修补师 | 修理知识 | 修车效率提高 |

---

### 4.2 数值被动固定生效

数值被动原则上**固定生效**，不再额外叠“是否触发”的概率。

原因：

> 权重修正本身已经是概率影响。  
> 如果再叠一层“这个被动有没有触发”，结果会太飘，玩家也很难感知角色价值。

正确流程：

```text
事件进入结算
→ 检查随行角色
→ 如果角色具备相关数值被动
→ 直接修正对应权重 / 信息 / 数值
```

不采用以下流程：

```text
事件进入结算
→ 先判定角色被动是否触发
→ 触发后再修正权重
```

---

### 4.3 数值被动叠加规则

| 情况 | 规则 |
|---|---|
| 同一角色的两个数值被动 | 可以同时生效，但通常作用于不同对象 |
| 多名角色影响同一结果 | 后续可设上限，避免无限叠加 |
| 同类权重修正 | 暂时不做复杂叠乘，后续整合时统一平衡 |
| 初始三人阶段 | 以单角色单效果为主，避免多重叠加 |

建议临时上限：

| 修正对象 | 建议上限 |
|---|---|
| 降低坏结果权重 | 最多降低到原始权重的 60% |
| 提高好结果权重 | 最多提高到原始权重的 150% |
| 信息等级 | 最高不超过 5 |
| 初始无装备、无情报、无记忆 | 免费信息最高到 2 |

## 五、事件 / 世界接口规则

### 5.1 事件接口是什么

事件接口不是常驻主动技能。  
它只在满足条件的事件中出现。

它可以表现为：

| 接口形式 | 说明 |
|---|---|
| 新增选项 | 事件里多一个角色相关选择 |
| 标记选项 | 给已有选项加上“护路人建议 / 射手判断 / 修补师判断” |
| 替换选项 | 把一个普通选项替换成更专业的选项 |
| 风险提示 | 在选择前显示更准确信息 |
| 结果修正 | 选择某个选项后，降低坏结果或提高好结果 |
| 路线提示 | 提前看出风险、痕迹、结构问题 |
| 聚落提示 | 看出市场、治安、医疗、维修异常 |

---

### 5.2 事件接口不是主动技能

事件接口不做成常驻按钮。

不应该出现：

```text
点击护路人技能：巡逻
点击射手技能：侦察
点击修补师技能：诊断
```

正确表现是：

```text
当事件发生时，如果队伍中有对应角色，就在事件界面中出现相关提示或选项。
```

例子：

```text
事件：夜里货车旁有人影

普通选项：
1. 大声驱赶
2. 假装没看见
3. 花钱请守夜人

如果队伍中有护路人：
4. 【护路人】让护路人守在货车旁，先观察对方动向
```

---

### 5.3 事件接口出现条件

事件接口需要满足事件标签。

| 事件标签 | 可能触发接口角色 |
|---|---|
| 敌对 / 夜袭 / 威胁 | 护路人 |
| 护送 / 抢货 / 车队保护 | 护路人 |
| 路线 / 痕迹 / 伏击 / 野兽 | 荒野射手 |
| 资源 / 狩猎 / 采集 | 荒野射手 |
| 车损 / 道路障碍 / 结构故障 | 修补师 |
| 医疗 / 伤员 / 中毒 / 污染 | 修补师 |
| 聚落治安 | 护路人 |
| 聚落谣言 | 荒野射手 |
| 聚落维修 | 修补师 |
| 聚落医疗 | 修补师 |

---

### 5.4 事件接口强弱控制

事件接口按强弱分三档：

| 接口强度 | 出现方式 |
|---|---|
| 弱接口 | 可以固定出现 |
| 中接口 | 可以固定出现，但收益有限 |
| 强接口 | 需要条件限制，或只降低风险，不直接给高收益 |
| 剧情关键接口 | 不随机，按剧情 / 支线 / 主线条件出现 |

当前基础版建议：

> 初始三人的事件接口以弱接口和中接口为主。  
> 不直接给大收益，不直接跳过核心风险。  
> 主要提供更稳的选择、更清楚的信息、更少的损失。

---

### 5.5 单个事件最多显示几个角色接口

为了避免事件界面太乱：

| 情况 | 规则 |
|---|---|
| 普通事件 | 最多显示 1 个角色接口 |
| 中大型事件 | 最多显示 2 个角色接口 |
| 主线 / 支线事件 | 可按剧情需要显示多个，后续再定 |

如果多个角色都符合条件，优先级按事件主题决定：

| 事件主题 | 优先角色 |
|---|---|
| 敌对威胁 | 护路人 |
| 路线痕迹 | 荒野射手 |
| 车损结构 | 修补师 |
| 医疗伤病 | 修补师 |
| 商队内部恐慌 | 护路人 |
| 聚落维修 | 修补师 |
| 聚落情报 | 荒野射手 |
| 聚落小治安 | 护路人 |

## 六、信息等级系统

### 6.1 信息等级 0–5

信息等级沿用旧版 6.3。

| 信息等级 | 名称 | 玩家看到什么 |
|---:|---|---|
| 0 | 无信息 | 不知道前方是什么 |
| 1 | 模糊信息 | 感觉前方有异常 |
| 2 | 类型信息 | 知道大概类型，比如野兽 / 强盗 / 道路问题 |
| 3 | 关键信息 | 知道主要风险点，比如伏击、车损、毒性、货损 |
| 4 | 详细信息 | 知道更明确的后果范围、敌人倾向、资源代价 |
| 5 | 精确信息 | 接近完整情报，可做最优准备 |

---

### 6.2 信息等级计算

```text
最终信息等级 =
事件基础信息
+ 区域信息
+ 角色数值被动
+ 角色事件接口
+ 工具 / 装备
+ 路线情报
+ 记忆 / 遗迹线索
```

最高不超过 5。

---

### 6.3 初始信息上限

初始三人无装备、无工具、无记忆、无额外情报时：

| 情况 | 上限 |
|---|---:|
| 普通隐藏 / 预警类事件 | 最高到 2 |
| 明显事件 | 可直接显示 2 |
| 需要专业工具的事件 | 初始角色不能直接到 3 |
| 高级遗迹 / 灰兽潮核心信息 | 初始角色不能直接看透 |

也就是说：

> 初始角色可以让玩家“看出类型”，但不能免费“看穿全部”。

## 七、初始三人能力槽

新版 6.3 中，初始三人每人 4 个能力槽。

---

### 7.1 护路人能力槽

| 槽位 | 类型 | 名称 | 核心作用 |
|---:|---|---|---|
| 1 | 数值被动 | 护卫 | 敌对、夜袭、护送、恐慌类事件中，最差混乱结果权重降低 |
| 2 | 数值被动 | 护车 | 货损、抢货、撤退中，订单货物、补给、零件损失权重降低 |
| 3 | 事件接口 | 挡在前面 | 在威胁、勒索、夜袭、护送事件中，提供更稳的正面处理方式 |
| 4 | 事件接口 | 维持秩序 | 在内部恐慌、聚落小治安、商队冲突事件中，降低局面失控风险 |

#### 护卫

适用范围：

- 强盗事件；
- 夜袭事件；
- 护送事件；
- 敌对人类事件；
- 内部恐慌事件；
- 聚落小治安事件。

效果：

```text
敌对、夜袭、护送、内部恐慌类事件中，
最差混乱结果权重 ×0.8。
```

说明：

> 护路人不能让危险消失，但能降低局面直接崩坏的概率。

#### 护车

适用范围：

- 货损事件；
- 抢货事件；
- 撤退结算；
- 车队保护；
- 订单货物受损；
- 补给 / 零件被夺。

效果：

```text
货损、抢货、撤退中，
订单货物、补给、零件损失权重 ×0.85。
```

说明：

> 护路人不是修车角色，但他懂得护住货车和货物。

#### 挡在前面

接口定位：

> 护路人在外部威胁面前提供正面保护选项。

可能出现的事件：

| 事件类型 | 接口表现 |
|---|---|
| 强盗勒索 | 增加“让护路人出面谈判 / 压阵”选项 |
| 夜袭 | 增加“护路人守住货车”选项 |
| 护送冲突 | 增加“护路人先挡住冲突”选项 |
| 聚落小治安 | 增加“护路人盯住货车附近的人”选项 |

效果倾向：

| 效果 | 说明 |
|---|---|
| 降低货损 | 不一定避免战斗 |
| 降低士气损失 | 商队更安心 |
| 降低最差结果 | 但可能消耗时间 / 士气 / 少量资源 |
| 提供稳妥方案 | 不是最高收益方案 |

#### 维持秩序

接口定位：

> 护路人帮助商队在混乱、恐慌、争执中保持秩序。

可能出现的事件：

| 事件类型 | 接口表现 |
|---|---|
| 内部争吵 | 增加“护路人压住局面”选项 |
| 士气恐慌 | 增加“护路人组织守夜 / 分工”选项 |
| 聚落骚乱 | 增加“护路人带队避开混乱区域”选项 |
| 商队冲突 | 增加“护路人划清边界”选项 |

效果倾向：

| 效果 | 说明 |
|---|---|
| 降低士气损失 | 但不一定完全避免 |
| 降低内部事件恶化 | 适合压力高时 |
| 降低失控结果 | 不直接给资源 |
| 增强队伍安全感 | 偏稳定，不偏收益 |

---

### 7.2 荒野射手能力槽

| 槽位 | 类型 | 名称 | 核心作用 |
|---:|---|---|---|
| 1 | 数值被动 | 野外侦察 | 路线、敌情、痕迹类事件信息等级 +1 |
| 2 | 数值被动 | 狩猎采集 | 自然资源事件中，补给类结果权重提高 |
| 3 | 事件接口 | 看懂痕迹 | 在路线、伏击、野兽、灰烬兽痕迹事件中，提供更准确信息或预警 |
| 4 | 事件接口 | 野外取食 | 在资源、狩猎、水源、采集事件中，提供更稳的资源获取方式 |

#### 野外侦察

适用范围：

- 路线事件；
- 敌情事件；
- 痕迹事件；
- 伏击预警；
- 野兽活动；
- 灰烬兽痕迹；
- 雾天 / 林地 / 河谷等视野受限区域。

效果：

```text
路线、敌情、痕迹类事件的信息等级 +1。
初始免费最高到 2。
```

说明：

> 射手可以看出“这里不对劲”，但不能免费知道所有细节。

#### 狩猎采集

适用范围：

- 自然资源事件；
- 野外采集；
- 水源事件；
- 小型猎物；
- 林地 / 河谷 / 残田牧野等资源区。

效果：

```text
自然资源事件中，
补给类结果权重 ×1.2。
```

说明：

> 荒野射手能提高找到食物和可用资源的机会，但不能凭空造补给。

#### 看懂痕迹

接口定位：

> 射手通过脚印、车辙、兽迹、灰烬痕迹判断前方风险。

可能出现的事件：

| 事件类型 | 接口表现 |
|---|---|
| 路线上有痕迹 | 增加“让射手判断痕迹” |
| 雾天前进 | 增加“射手先确认方向和动静” |
| 野兽活动 | 提示可能是普通野兽还是灰烬兽 |
| 伏击风险 | 提供更早预警 |
| 下雨后脚印 / 车辙 | 提供特殊判断信息 |

效果倾向：

| 效果 | 说明 |
|---|---|
| 提高信息等级 | 常见效果 |
| 降低伏击概率 | 但不完全免疫 |
| 提前知道敌人类型 | 方便准备 |
| 发现绕行机会 | 可能多耗时间或补给 |

#### 野外取食

接口定位：

> 射手在野外资源事件中提供更专业的采集、狩猎和判断。

可能出现的事件：

| 事件类型 | 接口表现 |
|---|---|
| 发现野果 / 菌类 | 判断是否安全 |
| 发现水源 | 判断是否可饮用 |
| 发现小型猎物 | 尝试狩猎 |
| 发现资源点 | 选择更稳的采集方式 |
| 暴雨后痕迹 | 判断是否值得追踪 |

效果倾向：

| 效果 | 说明 |
|---|---|
| 提高补给收益 | 但可能耗时 |
| 降低中毒 / 误采风险 | 尤其在湿地、林地 |
| 提供保守采集选项 | 收益低但风险小 |
| 提供冒险追踪选项 | 收益高但可能遇敌 |

### 7.3 修补师能力槽

| 槽位 | 类型 | 名称 | 核心作用 |
|---:|---|---|---|
| 1 | 数值被动 | 修理知识 | 修车效率提高；无零件临修效果提高 |
| 2 | 数值被动 | 现场诊断 | 结构、车损、普通设备故障信息等级 +1 |
| 3 | 事件接口 | 临场修补 | 在车损、道路障碍、维修事件中，提供临时处理方式 |
| 4 | 事件接口 | 伤病判断 | 在伤员、中毒、污染、聚落医疗事件中，提供更准确判断或低风险处理 |

#### 修理知识

适用范围：

- 车队维修；
- 临时修车；
- 车损事件；
- 零件不足时的应急处理。

效果：

```text
1 零件修车 +16。
无零件临时修车 +5。
```

说明：

> 修补师是当前队伍里最直接影响车况恢复的角色。

#### 现场诊断

适用范围：

- 结构问题；
- 车损问题；
- 道路障碍；
- 普通设备故障；
- 旧文明普通装置；
- 聚落维修服务异常。

效果：

```text
结构、车损、普通设备故障类事件信息等级 +1。
初始免费最高到 2。
```

说明：

> 修补师能判断“大概坏在哪里”，但不能免费破解高级旧文明设备。

#### 临场修补

接口定位：

> 修补师在车损、道路障碍、维修服务波动中提供临时处理方式。

可能出现的事件：

| 事件类型 | 接口表现 |
|---|---|
| 道路损坏 | 增加“让修补师检查车轴 / 车轮” |
| 车体受损 | 增加“临时加固车体” |
| 没有零件 | 增加“用废料临时补一下” |
| 聚落修车涨价 | 增加“修补师判断是否值得修” |
| 维修排队 | 增加“修补师先做基础处理” |

效果倾向：

| 效果 | 说明 |
|---|---|
| 降低车损 | 不一定完全修好 |
| 降低维修费用 | 可能只适用于小修 |
| 降低修车耗时 | 不替代城市工坊 |
| 提供临时方案 | 之后仍可能需要正式维修 |

#### 伤病判断

接口定位：

> 修补师在伤员、毒性、污染、医疗服务事件中提供更准确判断。

可能出现的事件：

| 事件类型 | 接口表现 |
|---|---|
| 队员受伤 | 增加“修补师先判断伤势” |
| 中毒风险 | 提示毒性来源或处理方式 |
| 灰烬污染 | 判断是否需要立刻处理 |
| 聚落医疗涨价 | 判断是否必须现在治疗 |
| 医馆排队 | 提供临时护理方案 |

效果倾向：

| 效果 | 说明 |
|---|---|
| 降低伤情恶化 | 但不免费治好 |
| 降低误治风险 | 尤其是中毒 / 污染 |
| 提供是否等待的判断 | 适合聚落医疗波动 |
| 降低轻伤处理成本 | 但重伤仍需城市治疗 |

### 10.1 能力槽是否升级

基础能力槽本身暂时不做经验升级。

也就是说：

| 内容 | 当前口径 |
|---|---|
| 护卫是否从 1 级升 2 级 | 暂不做 |
| 野外侦察是否刷熟练度 | 暂不做 |
| 修理知识是否靠使用次数升级 | 暂不做 |
| 事件接口是否升级 | 暂不做 |

后续成长通过：

- 战斗技能成长；
- 记忆；
- 装备；
- 工具；
- 职业变体；
- 城市设施；
- 支线奖励；

来体现。

### 10.2 为什么基础能力槽不升级

| 原因 | 说明 |
|---|---|
| 避免系统过重 | 角色已经有战斗技能成长 |
| 避免刷熟练度 | 不希望玩家为了升级反复刷事件 |
| 避免数值失控 | 权重类效果如果升级，很容易破坏概率 |
| 给装备 / 记忆留空间 | 高级能力由其他系统提供 |

## 十四、当前收口口径

6.3 v0.2 当前收口为：

> 每名角色拥有 4 个基础能力槽：2 个数值被动 + 2 个事件 / 世界接口。  
> 数值被动固定生效，用于修正事件权重、信息等级、资源收益、车损、货损、修车效率等，不再额外叠触发概率。  
> 事件接口不是常驻主动技能，而是在对应事件中提供额外选项、专业提示、替换选项或结果修正。  
> 初始三人保留旧版 6.3 的 2 个数值被动，并新增 2 个事件接口：护路人为“挡在前面 / 维持秩序”，荒野射手为“看懂痕迹 / 野外取食”，修补师为“临场修补 / 伤病判断”。  
> 初始角色免费信息最高到 2 级，3–5 级留给工具、装备、记忆、路线情报、后续角色和高级系统。  
> 6.3 不做战斗技能成长，不做能力槽升级，不做完整事件文本；这些分别留给 6.4、事件系统、职业变体、装备工具和支线系统。
