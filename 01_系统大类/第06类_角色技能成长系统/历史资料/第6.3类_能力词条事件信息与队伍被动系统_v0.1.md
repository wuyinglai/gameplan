# 《余烬商队》第 6.3 类：能力词条、事件信息与队伍被动系统 v0.1

## 一、6.3 的定位

第 6.3 类用于定义角色在远行、随机事件、探索、货损、撤退、资源结算和信息判断中的**固定被动专业性**。

6.1 解决"初始三人怎么战斗"。  
6.2 解决"重伤、记忆、招募、商队扩编"。  
6.3 解决"角色在非战斗系统里如何体现专业性"。

本版 6.3 不再采用"每人四个词条""主效果 / 次要效果""词条解锁主动选项"的设计。当前统一改为：

> 每名角色固定 2 个基础词条；每个词条只有 1 个效果；词条永远是被动，不是技能，不新增按钮，不新增主动选项。

---

## 二、6.3 当前硬规则

| 规则 | 当前口径 |
|---|---|
| 每名角色基础词条数量 | 固定 2 个 |
| 词条是否会变化 | 不会。角色是谁，就永远有这两个基础词条 |
| 词条是否升级 | 不靠经验、不靠熟练度升级 |
| 词条是否替换 | 不替换。记忆、装备、工具可以变化，但基础词条不变 |
| 词条是否主动 | 绝对不是主动技能 |
| 词条是否新增选项 | 不新增主动按钮，不新增可点击技能，不新增事件按钮 |
| 词条作用 | 只作为被动影响事件信息、随机权重、修车效率等 |
| 后续成长来源 | 记忆、装备、工具、城市准备、路线情报、新招募角色 |
| 信息类词条初始上限 | 初始角色最多把隐藏 / 预警类事件推到 2 级信息 |
| 权重类效果 | 直接调整权重，不再另加触发概率 |
| 数值类效果 | 必须小、固定、清楚，不能破坏核心系统 |

一句话：

> 词条不是技能。词条只是角色固定专业带来的被动修正。

---

## 三、词条生效条件

| 队员状态 | 基础词条是否生效 |
|---|---|
| 本趟随行，状态正常 | 生效 |
| 本趟随行，但已重伤退出 / 重伤恢复中 | 不生效 |
| 失能 | 不生效 |
| 留在城市的后备队员 | 不生效 |
| 已招募但本趟未带出门 | 不生效 |

正式口径：

> 只有本趟随行且可行动的队员，才提供基础词条。后备、重伤恢复中、失能队员不提供词条。

---

## 四、信息等级系统

信息采用 0–5 级。

| 信息等级 | 名称 | 玩家知道什么 | 示例文本 |
|---:|---|---|---|
| 0 | 无信息 | 什么都不知道，只知道有异常 | "前方出现异常。" |
| 1 | 模糊信息 | 知道可能有危险，但不知道类型 | "前方情况不太对，可能有危险。" |
| 2 | 类型信息 | 知道大概类型：敌人、资源、道路、污染、车损等 | "前方像是有人活动过，可能是敌对风险。" |
| 3 | 关键信息 | 知道具体类别：强盗、野兽、灰烬兽、断桥、资源点等 | "前方大概率是强盗活动痕迹。" |
| 4 | 详细信息 | 知道大致数量、方向、位置范围、后果范围 | "约 5–8 人，位置大概在北侧断路附近。" |
| 5 | 精确信息 | 知道具体人数、具体格子、具体触发条件 | "强盗 6 人，埋伏在前方第 3 格右侧废车后。" |

### 4.1 信息计算方式

信息不是单纯靠角色词条给，而是由多个来源相加。

```text
最终信息等级 = 事件基础信息 + 区域信息 + 角色词条信息 + 装备信息 + 路线情报 / 记忆信息
最高不超过 5
```

| 信息来源 | 范围 | 说明 |
|---|---:|---|
| 事件基础信息 | 0–2 | 事件本身给出的可见程度 |
| 区域信息 | 0–1 | 该区域是否本来就有明显特征 |
| 角色词条信息 | 0–1 | 随行角色是否有对应信息词条 |
| 装备 / 工具信息 | 0–1 | 是否有望远镜、探测器、地图、专用工具等 |
| 记忆 / 路线情报信息 | 0–2 | 旧文明记忆、驿站情报、商队传闻、城市情报等 |

### 4.2 初始阶段信息上限

为了避免开局信息过满，定一条限制：

```text
在没有装备、记忆、路线情报的初始阶段，隐藏 / 预警类事件的最终信息等级最高为 2。
```

| 阶段 | 信息上限 |
|---|---:|
| 初始三人，无装备 / 无记忆 / 无情报 | 最高 2 级 |
| 加上主动路线情报、驿站预警、基础工具 | 可到 3 级 |
| 中后期专用装备、专精角色、特殊记忆 | 可到 4 级 |
| 高级记忆、高级设备、剧情关键情报 | 可到 5 级 |

说明：明牌事件不受这个限制。比如敌人已经站在地图上，当然可以直接显示敌人。这个上限主要管"隐藏风险 / 预警事件"。

### 4.3 信息类型

不同角色只影响自己的专业信息，不是所有信息都加。

| 信息类型 | 说明 | 初始角色来源 |
|---|---|---|
| 路线信息 | 前路、岔路、雾天、危险地形 | 荒野射手【野外侦察】 |
| 敌情信息 | 强盗、野兽、灰烬兽、伏击、夜袭 | 荒野射手【野外侦察】 |
| 资源信息 | 补给、水源、草药、材料、猎物 | 荒野射手【狩猎采集】偏结果权重，不直接加信息 |
| 结构信息 | 桥梁、车轴、道路、坡道、废墟结构 | 修补师【现场诊断】 |
| 车损信息 | 车轴、车厢、车轮、抛锚隐患 | 修补师【现场诊断】 |
| 异常 / 旧文明信息 | 灰烬、火种、旧文明设备 | 初始修补师暂不专精，只能覆盖普通设备故障，不覆盖高级旧文明识别 |

护路人不提供信息等级。护路人负责稳定结算，不负责侦察。

## 五、初始三人固定词条

## 5.1 护路人

### 词条 1：【护卫】

| 项目 | 内容 |
|---|---|
| 词条效果 | 敌对、夜袭、护送、内部恐慌类事件中，"完全无防备 / 完全混乱"类最差结果的权重降低 |
| 是否主动 | 否，被动 |
| 是否概率触发 | 否。因为它本身就是权重调整 |
| 权重调整 | 最差结果权重 ×0.8 |
| 限制 | 不减少敌人数量，不降低敌人伤害，不减少买路钱，不取消战斗，不自动减士气 |

#### 例子：强盗事件结果池

如果某个强盗事件结果池原本是：

| 结果 | 原权重 |
|---|---:|
| 普通勒索 | 50 |
| 直接开战 | 30 |
| 完全混乱，被动受损 | 20 |

有【护卫】后，只调整最差结果：

| 结果 | 调整后权重 |
|---|---:|
| 普通勒索 | 50 |
| 直接开战 | 30 |
| 完全混乱，被动受损 | 16 |

然后系统再重新归一化抽取。

它不是免灾，只是让"最恶心的无防备结果"少一点。

---

### 词条 2：【护车】

| 项目 | 内容 |
|---|---|
| 词条效果 | 货损、抢货、撤退结算中，订单货物、补给、零件被随机损失的权重降低 |
| 是否主动 | 否，被动 |
| 是否概率触发 | 否。因为它本身就是权重调整 |
| 权重调整 | 订单货物、补给、零件的损失权重 ×0.85 |
| 限制 | 不减少总货损，不减少车损，不保证关键物资安全 |

#### 例子：货损结算

如果货损事件要随机损失 8 格货物，损失总量仍然是 8 格。

变化只是：订单货物、补给、零件被抽中的概率略微降低；普通货物、低价值货物相对更容易被抽中。

| 物资类型 | 原损失权重 | 有【护车】后 |
|---|---:|---:|
| 订单货物 | 100 | 85 |
| 补给 | 100 | 85 |
| 零件 | 100 | 85 |
| 普通货物 | 100 | 100 |
| 低价值战利品 | 100 | 100 |

---

## 5.2 荒野射手

### 词条 1：【野外侦察】

| 项目 | 内容 |
|---|---|
| 词条效果 | 路线、敌情、痕迹类事件的信息等级 +1 |
| 是否主动 | 否，被动 |
| 是否概率触发 | 否，固定生效 |
| 信息上限 | 初始阶段免费最高到 2 级 |
| 限制 | 不给 3 级以上信息，不降低遇敌率，不避免伏击，不判断具体人数和格子 |

#### 例子：前方车辙混乱

| 条件 | 信息等级 | 玩家看到 |
|---|---:|---|
| 无【野外侦察】 | 1 | 前方路面有些混乱，可能有危险。 |
| 有【野外侦察】 | 2 | 前方有车辙和脚印，像是有人活动过，可能是敌对风险。 |
| 后期有追踪记忆 / 工具 / 情报 | 3+ | 才可能知道是强盗、数量、方向、位置等。 |

---

### 词条 2：【狩猎采集】

| 项目 | 内容 |
|---|---|
| 词条效果 | 自然资源事件中，补给类结果权重小幅提高 |
| 是否主动 | 否，被动 |
| 是否概率触发 | 否。因为它本身就是权重调整 |
| 权重调整 | 补给类结果权重 ×1.2 |
| 限制 | 不降低材料类结果权重，不新增资源事件，不稳定白送补给，不增加补给数量上限 |

#### 例子：自然资源事件结果池

如果自然资源事件结果池原本是：

| 结果 | 原权重 |
|---|---:|
| 获得材料 | 40 |
| 获得补给 | 30 |
| 获得少量金币 / 杂物 | 20 |
| 什么都没有 | 10 |

有【狩猎采集】后：

| 结果 | 调整后权重 |
|---|---:|
| 获得材料 | 40 |
| 获得补给 | 36 |
| 获得少量金币 / 杂物 | 20 |
| 什么都没有 | 10 |

它只是让自然资源更偏向"能吃的东西"，不会凭空多给资源。

---

## 5.3 修补师

### 词条 1：【修理知识】

| 项目 | 内容 |
|---|---|
| 词条效果 | 修补师参与修车时，修车效率提高 |
| 是否主动 | 否，被动，作用于已有修车行为 |
| 是否概率触发 | 否，固定生效 |
| 限制 | 不自动减少事件车损，不免费修车，不替代城市维修 |

#### 修车规则

| 修车方式 | 普通角色 | 有【修理知识】 |
|---|---:|---:|
| 消耗 1 零件修车 | +10 车队耐久 | +16 车队耐久 |
| 无零件临时修车 | +3 车队耐久 | +5 车队耐久 |

这是目前初始三人里最明确的数值专业。它可以固定，因为它不自动免损，只是同样修车时修得更好。

---

### 词条 2：【现场诊断】

| 项目 | 内容 |
|---|---|
| 词条效果 | 结构、车损、普通设备故障类事件的信息等级 +1 |
| 是否主动 | 否，被动 |
| 是否概率触发 | 否，固定生效 |
| 信息上限 | 初始阶段免费最高到 2 级 |
| 限制 | 不包含敌情侦察，不包含高级旧文明识别，不治疗伤员，不自动减少车损 |

#### 例子：断桥

| 条件 | 信息等级 | 玩家看到 |
|---|---:|---|
| 无【现场诊断】 | 1 | 前方桥面受损，看起来不太安全。 |
| 有【现场诊断】 | 2 | 桥面承重可能不足，强行通过可能造成车损。 |
| 后期有工程工具 / 专精角色 / 记忆 | 3+ | 才可能知道具体加固点、损失范围、安全通过方案。 |

## 七、明确不做内容

| 不做内容 | 原因 |
|---|---|
| 不做每人 4 个基础词条 | 初始角色会过于全能，后期成长空间不足 |
| 不做主效果 / 次要效果 | 容易写成技能说明，复杂度失控 |
| 不做词条主动选项 | 词条必须是被动，主动选项属于事件、技能、道具或记忆 |
| 不做"护卫站位 / 有序撤退 / 主动调停"等词条按钮 | 这些会把词条变成技能 |
| 不做权重调整再叠概率触发 | 权重调整本身已经是概率影响，再叠触发概率会太飘 |
| 不让护路人提供信息等级 | 护路人不是侦察角色 |
| 不让射手同时拥有侦察和追踪两个信息词条 | 初始射手不能吃掉后续向导 / 猎人 / 记忆空间 |
| 不让修补师覆盖医护和高级旧文明识别 | 医护、旧文明专精留给后续角色或记忆 |
| 不让词条自动降低士气损失 | 士气 -1 已是中等事件，不能被常驻被动抵消 |
| 不让词条自动减少总货损 | 护车只降低关键物资被抽中权重，不减少总损失 |

## 八、与 6.2 和车厢容量的关系

第 6.2 类已经确定：商队扩编和多带人主要通过补给消耗、车厢空间、远行距离形成软限制。

6.3 的词条系统要服务于这个结构：

| 系统 | 关系 |
|---|---|
| 车厢容量 | 决定能带多少补给和货物 |
| 补给消耗 | 决定能带多少人跑多远 |
| 随行角色 | 决定本趟有哪些固定词条生效 |
| 词条 | 让本趟远行在信息、权重、修车上稍微更专业 |
| 记忆 / 装备 / 工具 | 提供后期成长，不改变基础词条 |

初始车队只适合 3 人短程远行。  
后续车厢容量、装备、记忆、新角色扩展后，玩家才逐步获得更高信息等级和更强远行稳定性。

---

## 九、总结

6.3 当前口径收束为：

> 每个角色固定 2 个被动词条；每个词条只有 1 个效果；词条不主动、不升级、不替换；初始信息最多到 2 级；权重类效果直接调整权重，不再叠概率；后续成长交给记忆、装备、工具、路线情报和新角色。

这版的目标不是让初始三人变强，而是让他们在远行和事件中体现一点清楚、稳定、可维护的专业性。
