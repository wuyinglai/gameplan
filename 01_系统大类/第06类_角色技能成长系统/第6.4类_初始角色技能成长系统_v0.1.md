## 1. 本版已修正内容

| 项目 | 当前口径 |
|---|---|

## 4. 技能与记忆适配

技能可以有职业标签，记忆也可以有职业标签。

例：

| 对象 | 标签 |
|---|---|
| 横身拦截 | 护卫 |
| 重击 | 当前按护卫处理 |
| 射箭 | 射手 |
| 兽迹判断 | 猎手 |
| 简易治疗 | 医师 |
| 车体修理 | 工匠 |
| 灰烬净化 | 医师、灰烬 |
| 治疗箭记忆 | 射手、医师 |
| 车厢盾架记忆 | 护卫、工匠 |

适配规则：

```text
如果 记忆标签 ∩ 技能标签 ≠ 空，
则该记忆可以用于该技能成长。
```

例：

| 技能 | 记忆 | 是否适配 |
|---|---|---|
| 射箭【射手】 | 旧弩手记忆【射手】 | 适配 |
| 射箭【射手】 | 伤口缝合记忆【医师】 | 不适配 |
| 治疗箭【射手、医师】 | 旧弩手记忆【射手】 | 适配 |
| 治疗箭【射手、医师】 | 伤口缝合记忆【医师】 | 适配 |
| 车厢盾架【护卫、工匠】 | 旧盾墙记忆【护卫】 | 适配 |
| 车厢盾架【护卫、工匠】 | 废车工坊记忆【工匠】 | 适配 |

## 5. 记忆可以产生主动技和被动技

记忆不是单纯材料。记忆可以通过融合、设施处理或火种引导，转化为角色可用的技艺。

```text
记忆技艺
├─ 主动技
└─ 被动技
```

主动技：

| 类型 | 说明 |
|---|---|
| 攻击技 | 造成伤害，例如射箭、重击、放血箭 |
| 防御技 | 获得护盾、护车、防护，例如横身拦截、车队防护 |
| 辅助技 | 治疗、净化、标记、修车、加固等 |

被动技：

| 类型 | 说明 |
|---|---|
| 战斗被动 | 战斗中自动生效 |
| 远行被动 | 路线、天气、野外资源相关 |
| 行商被动 | 交易、议价、市场、订单相关 |
| 事件被动 | 野外事件、聚落事件、内部事件中生效 |

注意：

```text
主动 / 被动是技艺表现形式。
职业标签是记忆、设施、火种是否适配的规则。
两者不是一回事。
```

## 6. 技能成长两条线

技能成长分为两条互相独立的线：

```text
技能成长
├─ 普通升级：金币 + 设施，用于数值升级 / 状态升级
└─ 火种升级：火种 + 特殊设施，用于词条 / 后缀 / 特殊效果
```

关键规则：

| 项目 | 规则 |
|---|---|
| 普通升级 | 提升技能内部的数值项或状态项 |
| 火种升级 | 给技能添加词条 / 后缀 / 特殊效果 |
| 是否互相替代 | 不互相替代 |
| 高普通等级是否等于有火种词条 | 不等于 |
| 火种词条多是否等于普通等级高 | 不等于 |

例：

```text
一个普通等级很高的射箭，可以没有任何火种词条。
一个普通等级较低的射箭，也可以拥有火种词条。
```

---

## 7. 普通升级：数值升级与状态升级

普通升级不再只是“整张技能卡自动升一级”。

每次普通升级时，玩家可以在该技能允许的升级项中选择。

普通升级分为两类：

```text
普通升级
├─ 数值升级
└─ 状态升级
```

### 7.1 数值升级

数值升级负责直接增加数值。

| 类型 | 示例 |
|---|---|
| 伤害数值 | 伤害 +1 |
| 护盾数值 | 护盾 +1 |
| 治疗数值 | 治疗 +1 |
| 修车数值 | 修车 +1 |
| 车体防护 | 车体防护 +1 |

数值升级相对基础，价格应低于状态升级，但具体价格本版不定。

### 7.2 状态升级

状态升级负责让技能施加的正面 / 负面状态更强。

| 类型 | 示例 |
|---|---|
| 负面状态层数 | 标记层数 +1 |
| 负面状态层数 | 流血层数 +1 |
| 负面状态层数 | 中毒层数 +1 |
| 正面状态强度 | 护盾强度 +1 |
| 正面状态层数 | 防护层数 +1 |
| 净化处理 | 净化层数 +1 |

注意：

```text
不写“强化标记”。
要写成具体规则，例如“标记层数 +1”。
```

状态升级比数值升级更贵，但具体价格后续再定。

---

## 8. 普通等级显示方式

普通等级可以作为 UI 上的总显示等级，但内部应拆成：

```text
普通总等级 = 数值升级点 + 状态升级点
```

例：

```text
精确射击 Lv6
= 数值升级点 5
+ 状态升级点 1
```

这样玩家看到的是一个清楚的等级，系统内部也能区分“伤害成长”和“状态成长”。

---

## 9. 技能融合规则

技能融合不设主技能 / 副技能。

多个技能或记忆可以融合成新技能 / 复合记忆。

融合时分别继承：

```text
新技能数值升级点 = 参与融合技能中最高的数值升级点
新技能状态升级点 = 参与融合技能中最高的状态升级点
新技能普通总等级 = 新技能数值升级点 + 新技能状态升级点
```

融合后，新技能允许玩家在新技能可用升级项中重新分配继承来的升级点。

---

## 10. 融合示例：射击 + 标记 = 精确射击

假设：

```text
射击
- 数值升级点：5
- 主要效果：造成伤害

标记
- 基础效果：施加 1 层标记
- 状态升级点：1
- 当前效果：施加 2 层标记
```

融合：

```text
射击 + 标记 → 精确射击
```

继承：

```text
精确射击
- 数值升级点：5
- 状态升级点：1
- 普通总等级：6
```

效果可以表现为：

```text
造成伤害，并施加 2 层标记。
```

其中：

| 来源 | 继承内容 |
|---|---|
| 射击 | 继承 5 点数值升级，可用于提高伤害 |
| 标记 | 继承 1 点状态升级，使标记从 1 层变成 2 层 |

这条规则的目的：

> 高等级伤害技能不会因为融合被清零；已经升级过的状态技能也不会浪费。

## 11. 火种升级与词条池

火种升级独立于普通等级。

火种负责：

| 作用 | 示例 |
|---|---|
| 添加词条 | 射箭【重击】 |
| 添加特殊效果 | 射箭【穿甲】 |
| 改变方向 | 治疗箭、车厢盾架 |
| 解锁特殊处理 | 治疗【净毒】、车体修理【加固】 |

火种词条不随机，也不是所有技能都能随便加。

采用规则：

```text
每个技能有自己的火种词条池。
玩家在已解锁、且该技能允许的词条池中自由选择。
```

火种词条可以多个，但成本递增。

例：

| 第几个火种词条 | 消耗倾向 |
|---:|---|
| 第 1 个 | 低 |
| 第 2 个 | 更高 |
| 第 3 个 | 很高 |
| 后续 | 继续递增，具体公式后续再定 |

---

## 12. 火种词条类型限制

火种词条需要类型限制，避免不合理组合。

| 词条类型 | 可用于 |
|---|---|
| 攻击词条 | 攻击技，例如射箭、重击、放血箭 |
| 防御词条 | 防御技，例如横身拦截、稳住阵线 |
| 治疗词条 | 治疗技，例如简易治疗 |
| 修补词条 | 修车 / 加固类技能 |
| 状态词条 | 标记、流血、中毒、净化等相关技能 |
| 行商词条 | 后续行商技能 |
| 灰烬词条 | 灰烬净化、污染、火种相关技能 |

规则：

```text
治疗类技能不能随便加【重击】。
攻击技、防御技、治疗技、修补技、状态技等，各自拥有自己的可选词条池。
```

---

## 13. 融合时火种词条处理

火种词条很稀有，融合时原则上尽量保留。

但必须经过新技能词条池检查。

| 情况 | 处理 |
|---|---|
| 新技能支持该词条 | 保留 |
| 新技能不支持该词条 | 不强行保留，可转为重选或暂存 |
| 多个融合材料都有火种词条 | 所有适配词条尽量保留，成本 / 规则后续平衡 |
| 词条类型明显不适配 | 不能硬塞进新技能 |

例：

```text
射箭【穿甲】 + 标记 → 精确射击
如果精确射击属于攻击技，并支持【穿甲】，则该词条可以保留。
```

```text
简易治疗 + 重击类词条
如果新技能是纯治疗技，则不能强行保留攻击词条。
```

## 17. 当前收口口径

6.4 当前采用以下口径：

> 初始角色技能成长分为普通升级和火种升级两条独立线。普通升级由金币和设施支持，分为数值升级与状态升级；数值升级包括伤害、护盾、治疗、修车等直接数值提升，状态升级包括标记层数、流血层数、正面状态强度、净化层数等效果提升。火种升级独立于普通等级，用于添加词条、后缀或特殊效果。技能融合不设主副，融合后的新技能分别继承参与技能中最高的数值升级点和最高的状态升级点，并允许玩家重新分配。火种词条原则上尽量保留，但必须通过新技能词条池检查。升级价格本版不定，后续在全局物价框架确定后再补。
