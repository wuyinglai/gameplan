# 第一部分：全局原则与定稿（6.1 + 6.2 全局部分）

---

## 1.1 第 6.1 的定位

第 6.1 类用于确定《余烬商队》初始三名队员的基础战斗结构。

本节主要解决：

1. 初始三名队员是谁；
2. 三名队员的生命值与能力评级；
3. 三名队员的职业定位；
4. 三名队员的基础技能牌、费用与前期数值；
5. 后期增强版技能的数值方向；
6. 护盾、护甲、屏障、穿甲、标记、流血、迟滞、盾反等核心战斗状态接口；
7. 普通战、精英战、Boss 战的节奏目标；
8. 敌人意图标签如何展示；
9. 后期高条件爆发技能的设计边界；
10. 旧版本中与当前口径冲突的内容如何废弃。

本节不处理后期招募、替补、记忆技能、火种成长、队员长期伤病与休养。这些内容放到后续 6.2、6.3 处理。

第 6.1 的目标是先把"初始三人 + 初始技能 + 基础战斗口径"定稳。后续所有第 6 类成长、伤病、远行能力，都要建立在这一节之上。

### 1.1.1 第 6.2 的定位

第 6.1 已经确定了初始三名队员、基础技能、生命值、护盾、护甲、屏障、穿甲、标记、流血、迟滞、灰蚀、过载等基础战斗接口。

第 6.2 要解决的是：

1. 队员在战斗中生命降到 0 或以下后怎么处理；
2. 重伤如何影响本场战斗和后续远行；
3. 重伤如何自然恢复、加速恢复、城市治疗；
4. 货车耐久归零和战斗终止如何判断；
5. 队员重伤后抽牌和专属牌如何变化；
6. 内伤、外伤、灰蚀、中毒如何与长期状态衔接；
7. 不做经验、不做熟练度后，角色怎么成长；
8. 旧文明"记忆"是什么，如何获得、装配、强化、融合；
9. 初始技能与后续记忆如何共享技能/记忆位；
10. 后续如何招募新队员；
11. 商队如何从小型出行扩展到中型、大型出行；
12. 上场人数增加后，行动力、抽牌、补给消耗如何变化；
13. 如何通过补给和车厢空间形成软限制，而不是硬性禁止玩家带人；
14. 哪些内容暂不做，避免系统爆炸。

一句话：
> 6.1 是"初始三人怎么打"。
> 6.2 是"他们打久了以后怎么受伤、怎么恢复、怎么获得旧文明记忆、怎么招募新人、怎么把商队做大"。

### 1.1.2 第 6.3 的定位

第 6.3 类用于解决：角色在战斗之外，如何影响野外事件、聚落事件、信息显示、路线判断、资源事件、车队损失、内部压力和世界互动。

它不是战斗技能系统。
战斗技能成长放到后续 6.4 初始角色技能成长系统中处理。

6.3 主要处理四件事：

| 模块 | 说明 |
|---|---|
| 能力槽结构 | 每名角色拥有 2 个数值被动 + 2 个事件/世界接口 |
| 信息等级 | 角色如何提高事件信息、路线信息、风险提示 |
| 权重修正 | 角色如何降低坏结果权重、提高好结果权重 |
| 事件接口 | 角色如何在事件中提供额外选项、替代选项、提示或特殊处理方式 |

一句话定位：
> **6.3 不是让角色多几个战斗技能，而是让角色在战斗外也有专业存在感。**

### 1.1.3 第 6.4 的定位

6.4 解决的是：初始三名角色的技能，后续如何成长。

它不处理：

| 不处理内容 | 所属系统 |
|---|---|
| 非战斗能力槽、事件接口 | 6.3 |
| 初始职业变体、同职业招募方向 | 6.5 |
| 城市设施完整建设 | 第 8 类 |
| 全局物价与订单奖励 | 后续经济系统/总数值表 |
| 代码实现 | 后续开发任务 |

一句话：6.4 只定"技能怎么长"，不定"每升一级多少钱"。

# 第二部分：能力评级标准

第 6.1 统一使用 A / B / C / D 四档评级。

| 等级 | 含义 |
|---|---|
| A | 高，是角色核心优势 |
| B | 中等，有稳定作用 |
| C | 有一点，但不多 |
| D | 基本没有/完全没有 |

不使用 A+、A-、B+、C- 等加减评级。评级只用于快速定位角色能力，不用于精确数值计算。

## 3.1 初始三人生命值

当前建议生命值如下：

| 角色 | 生命 | 定位 |
|---|---:|---|
| 护路人 | 68 | 主承伤、防护、护车、盾反 |
| 修补师 | 46 | 全能辅助、修车、加固、少量治疗 |
| 荒野射手 | 40 | 脆皮高伤害、标记、流血、侦察 |
| 合计 | 154 | 位于 130–170 合理范围内 |

### 3.1.1 护路人为什么是 68

护路人是初始队伍中最能扛的角色，必须让他的血量优势明显。护路人的职责包括：承受敌方直接伤害；给自己和队友提供护盾；保护货车；使用盾反惩罚攻击者；在危险遭遇中稳定队伍。如果护路人血量只比修补师高一点，他的主承伤定位会不明显。因此护路人生命定为 68。

### 3.1.2 修补师为什么是 46

修补师是全能辅助，不是前排，也不是半个坦克。修补师需要比射手略稳，因为他承担商队维护功能；但不能太高，否则会模糊护路人的主防御地位。修补师职责包括：修车；加固；少量治疗；提供临时护盾；在关键时刻补救队伍和货车状态。因此修补师生命定为 46。

### 3.1.3 荒野射手为什么是 40

荒野射手是脆皮高伤害角色。他的职责包括：高输出；标记敌人；施加流血；施加迟滞；补刀；远行中发现痕迹和敌情。射手生命最低，是为了让玩家明确感受到保护输出位的必要性。因此荒野射手生命定为 40。

## 3.2 初始三人能力评级表

| 角色 | 生存 | 输出 | 防护 | 治疗 | 修车 | 情报 | 状态处理 |
|---|---|---|---|---|---|---|---|
| 护路人 | A | B | A | D | D | D | D |
| 荒野射手 | C | A | D | D | D | A | C |
| 修补师 | B | D | B | B | A | D | D |

### 3.2.1 护路人评级说明

护路人是防护核心。

| 能力 | 评级 | 说明 |
|---|---|---|
| 生存 | A | 血量最高，负责承伤 |
| 输出 | B | 不是主输出，但有横身拦截、重击、盾反等补伤害手段 |
| 防护 | A | 核心能力，负责护盾、护车、稳阵 |
| 治疗 | D | 没有治疗能力 |
| 修车 | D | 不负责修车 |
| 情报 | D | 不负责侦察 |
| 状态处理 | D | 不负责清除异常状态 |

护路人的关键词是：撑住、保护、护车、盾反。他不是纯输出角色，但也不是完全没有伤害。

### 3.2.2 荒野射手评级说明

荒野射手是输出和情报核心。

| 能力 | 评级 | 说明 |
|---|---|---|
| 生存 | C | 脆，但不是完全不能挨打 |
| 输出 | A | 初始队伍最高输出角色 |
| 防护 | D | 不提供护盾或保护 |
| 治疗 | D | 没有治疗 |
| 修车 | D | 不负责修车 |
| 情报 | A | 远行中负责痕迹、伏击、敌情判断 |
| 状态处理 | C | 能施加标记、流血、迟滞等战斗状态，但不清除异常 |

荒野射手的关键词是：高伤害、标记、流血、补刀、侦察。他有很高的战斗价值，但需要护路人和修补师保护。

### 3.2.3 修补师评级说明

修补师是全能辅助。

| 能力 | 评级 | 说明 |
|---|---|---|
| 生存 | B | 比射手稳，但不是前排 |
| 输出 | D | 基本不承担输出职责 |
| 防护 | B | 可以提供护甲、护盾和货车保护，但不如护路人 |
| 治疗 | B | 有治疗，但治疗量收敛 |
| 修车 | A | 核心能力 |
| 情报 | D | 不承担侦察职责 |
| 状态处理 | D | 当前基础定位不做状态清除专家 |

修补师的关键词是：修车、加固、少量治疗、补救。他不是强奶妈，不是输出角色，也不是主防御角色。

### 3.2.4 伤害结算顺序

1. 屏障（高级防御）
2. 护盾（临时防御）
3. 护甲（长期防御）
4. 生命

## 3.3 战斗节奏

### 3.3.1 战斗回合参考

| 战斗类型 | 回合数 | 说明 |
|---|---|---|
| 普通小怪战 | 约 5 回合 | 日常遭遇，快速解决 |
| 较难普通战 | 6–8 回合 | 需要一定策略 |
| 精英战 | 约 10 回合 | 需要铺垫与资源管理 |
| Boss 战 | 通常 15 回合以上 | 强 Boss 可 20 回合以上 |

**设计原则：** 速杀流可以更快，但设计基准不能太短；Boss 战需要足够的回合来展开机制。

### 3.3.2 敌人数量

敌人数量不设硬上限，按敌人强弱决定。强敌可单个（如精英、Boss），弱敌如狼群可 4 个以上。

### 3.3.3 基础战斗结构

| 项目 | 数值/规则 |
|---|---|
| 每回合行动力 | 3 点，全队共享 |

### 3.3.4 战斗收益定位（关联第 2.1 类经济系统）

| 战斗类型 | 收益定位 |
|---|---|
| 普通战斗 | 能覆盖一部分消耗，有中等金币/物资收益 |
| 高风险普通战 | 金币、补给、零件、药包中至少有一类明显收益 |
| 精英战斗 | 应有明显收益，让玩家愿意承担风险 |
| Boss/关键战斗 | 可包含火种、路线解锁、订单保障、城市功能推进等特殊价值 |

### 3.3.5 收入结构（关联第 2.1 类经济系统）

| 收入来源 | 定位 | 收入强度 |
|---|---|---|
| 主订单 | 最稳定收入，路线主收益来源 | 高 |
| 顺路订单/中途订单 | 路线规划收益核心 | 中高 |
| 战斗收益 | 鼓励玩家愿意打，不让战斗变纯损耗 | 中到中高 |
| 精英/高风险战斗 | 高风险对应高价值 | 高 |
| 事件收益 | 补充收入，随事件风险变化 | 中 |
| 自营贸易 | 未来重要收入来源 | 中高 |
| 每回合抽牌 | 7 张 |
| 回合结束 | 弃掉未使用手牌（默认） |
| 站位 | UI 队列，不是棋盘 |
| 敌人排列 | 一维队列，用于燃烧扩散、连锁、相邻关系、逃跑等 |

# 第四部分：重伤与恢复系统（6.2 全量）

## 4.1 战斗中生命归零与重伤退出

队员生命值降到 0 或以下后，立刻进入重伤退出状态。

| 情况 | 结果 |
|---|---|
| 队员生命值降到 0 或以下 | 该队员立即退出本场战斗 |
| 重伤退出后 | 不能继续行动 |
| 重伤退出后 | 不能继续使用该队员专属牌 |
| 重伤退出后 | 后续抽牌不再抽取该队员的 2 张专属牌 |
| 战斗结束后 | 按重伤次数进入一次重伤、二次重伤或失能 |
| 是否死亡 | 否 |
| 是否可在本场被治疗拉起 | 否，重伤后本场直接退出 |

重伤不是"倒地后等待奶妈拉起来"。一旦重伤，这场战斗该角色就不能继续参与。这样可以避免战斗变成"倒下—拉起—再倒下—再拉起"的循环，也能让重伤有足够分量。

## 4.2 重伤后专属牌与动态抽牌

每名上场角色每回合提供 2 张专属牌。此外，每回合固定抽 1 张车队牌。

基础公式：
```text
每回合抽牌数 = 当前可行动队员数 × 2 + 1 张车队牌
```

重伤退出的队员：
1. 本场不能行动；
2. 不能使用该角色专属牌；
3. 从下一次抽牌开始，不再抽取该角色的 2 张专属牌；
4. 已在手牌中的该角色专属牌不可继续使用，可视为失效、灰掉或弃置；
5. 车队牌仍可抽取，前提是货车仍有可结算状态。

### 4.2.1 小型出行抽牌变化

小型出行满员为 3 人上场。

| 当前可行动队员 | 抽牌数 | 构成 |
|---|---:|---:|---|
| 3 人 | 7 张 | 6 张人员专属牌 + 1 张车队牌 |
| 2 人 | 5 张 | 4 张人员专属牌 + 1 张车队牌 |
| 1 人 | 3 张 | 2 张人员专属牌 + 1 张车队牌 |
| 0 人 | 1 张 | 只剩 1 张车队牌 |

例子：三人小队正常每回合抽 7 张牌。如果荒野射手重伤退出，则不再抽取荒野射手的 2 张专属牌。后续抽牌变为 5 张：护路人 2 张、修补师 2 张、车队牌 1 张。

## 4.3 战斗终止与强制撤退条件

本场战斗不能简单以"全员重伤"作为唯一终止条件。因为货车也是战斗核心单位，玩家仍然可能通过车队牌、货车牌或战斗内机制继续结算。

当前定稿：
> 只有当所有出战队员均重伤退出，且货车耐久归零时，才终止本场战斗，并触发强制撤退到最近安全点。

| 战斗中情况 | 是否终止战斗 | 说明 |
|---|---|---|
| 1 名队员重伤退出 | 否 | 剩余队员继续战斗 |
| 多名队员重伤退出，但仍有人可行动 | 否 | 剩余队员继续战斗 |
| 所有队员重伤退出，但货车耐久仍大于 0 | 不立即终止 | 仍可结算货车牌/车队牌 |
| 货车耐久归零，但仍有队员可行动 | 不直接终止 | 进入严重危机，仍可能战斗内处理 |
| 所有队员重伤退出 + 货车耐久归零 | 终止 | 强制撤退到最近安全点 |

正式句：战斗终止条件不是"全员重伤"本身，也不是"货车耐久归零"本身，而是两者同时满足：所有出战队员均重伤退出，且货车耐久归零。

## 4.4 货车耐久归零与抛锚

货车耐久归零的基础口径应直接写明，不再只写"承接旧表"。

当前口径：

| 情况 | 结果 |
|---|---|
| 货车耐久归零 | 进入抛锚状态 |
| 抛锚后 | 停在当前格，不能继续移动 |
| 抛锚后 | 不能继续订单路线 |
| 抛锚后 | 不能进行普通野外探索 |
| 修复要求 | 必须将货车耐久修到 20 以上才能继续移动 |
| 是否立刻失败 | 否 |
| 是否直接强制撤退 | 否 |
| 长期无法修复/补给崩溃/士气崩溃 | 进入撤退风险或强制撤退 |

6.2 中要强调：货车耐久 0 是极严重状态，但不是"游戏立刻失败"。它表示商队抛锚，必须修复到可移动状态。如果同时全员重伤退出，则本场战斗终止并强制撤退。

## 4.5 一次重伤、二次重伤、失能

重伤最多可承受两次。第三次进入失能。

| 次数 | 状态 | 自然恢复时间 | 恢复后生命 | 最大生命惩罚 |
|---|---:|---|---:|---:|---:|
| 第 1 次 | 一次重伤 | 3 天 | 当前最大生命 20%，向上取整 | 最大生命 -20% |
| 第 2 次 | 二次重伤 | 4 天 | 当前最大生命 20%，向上取整 | 最大生命累计 -50% |
| 第 3 次 | 失能 | 不自然恢复为可作战 | 需要城市治疗 | 不能上场作战 |

### 4.5.1 最大生命惩罚

最大生命惩罚按累计处理。

| 角色 | 原始生命 | 一次重伤后最大生命 | 二次重伤后最大生命 |
|---|---|---|---:|---:|
| 护路人 | 68 | 55 | 34 |
| 修补师 | 46 | 37 | 23 |
| 荒野射手 | 40 | 32 | 20 |

计算方式：
- 一次重伤后：原始最大生命 × 80%，向上取整；
- 二次重伤后：原始最大生命 × 50%，向上取整；
- 第三次重伤：失能，不能上场。

### 4.5.2 恢复后当前生命

重伤自然恢复后，当前生命恢复到**当前最大生命的 20%**，向上取整。

例子：

| 角色 | 状态 | 当前最大生命 | 自然恢复后当前生命 |
|---|---|---|---:|---:|
| 荒野射手 | 一次重伤后 | 32 | 7 |
| 荒野射手 | 二次重伤后 | 20 | 4 |
| 护路人 | 一次重伤后 | 55 | 11 |
| 护路人 | 二次重伤后 | 34 | 7 |

这样恢复后能继续行动，但状态很危险，需要玩家决定是否继续上路。

---

## 4.6 重伤自然恢复

重伤可以自然恢复。

| 状态 | 自然恢复 |
|---|---|
| 一次重伤 | 3 天后恢复为可出战 |
| 二次重伤 | 4 天后恢复为可出战 |
| 失能 | 不自然恢复为可出战 |

自然恢复只代表角色能再次上场，不代表伤根完全治好。最大生命惩罚仍然存在，除非通过城市治疗恢复。

自然恢复后的角色：
- 可以上场；
- 当前生命为当前最大生命的 20%，向上取整；
- 重伤次数仍记录；
- 最大生命惩罚仍保留。

---

## 4.7 驿站/城镇加速恢复

驿站和城镇不能治疗重伤次数，也不能恢复最大生命惩罚，但可以加速自然恢复。

| 地点 | 可以做什么 | 不能做什么 |
|---|---|---|
| 驿站 | 加速一次/二次重伤自然恢复 | 不能清除重伤次数 |
| 驿站 | 可将剩余恢复时间缩短，例如 3 天改为 1 天 | 不能恢复最大生命惩罚 |
| 城镇 | 加速一次/二次重伤自然恢复 | 不能治疗失能 |
| 城镇 | 提供更稳定休整 | 不能清除重伤惩罚 |

例子：护路人一次重伤，本来需要 3 天自然恢复。如果在驿站接受休整，可以花费金币和时间，将恢复时间缩短为 1 天。但恢复后仍然保留一次重伤记录和最大生命 -20% 惩罚。这让驿站和城镇有用，但不会取代城市治疗。

---

## 4.8 城市治疗与非永久惩罚原则

城市可以治疗一次重伤、二次重伤和失能。治疗统一花费 1 天，不再设置 3–5 天的超长治疗时间，也不需要特殊城市设施。

| 状态 | 城市治疗效果 | 时间 | 代价 |
|---|---|---|---:|---|
| 一次重伤 | 清除一次重伤记录，恢复最大生命惩罚 | 1 天 | 金币 |
| 二次重伤 | 清除二次重伤记录，恢复最大生命惩罚 | 1 天 | 更多金币 |
| 失能 | 清除失能，恢复为可出战状态 | 1 天 | 大量金币 |

原则：
> 正常游玩不应留下永久惩罚。
> 最大生命惩罚、重伤次数、失能，都可以通过城市治疗恢复。
> 除非是明确剧情选择或特殊事件，否则不要对玩家施加永久伤残。

原因：永久惩罚容易让玩家觉得系统恶心；随机战斗或天气导致永久损失，会强迫玩家读档；这类游戏压力应来自资源、时间、路线和选择，而不是不可逆伤残。

---

## 4.9 重伤恶化

重伤可以因恶劣天气或特殊环境发生恶化。但恶化不能升级重伤次数，也不能直接导致失能。

当前定稿：

| 触发 | 效果 |
|---|---|
| 伤员恶化 | 当天不减少重伤恢复时间 |
| 额外惩罚 | 重伤剩余恢复时间 +1 天 |
| 是否增加重伤次数 | 否 |
| 是否从一次重伤变二次重伤 | 否 |
| 是否从二次重伤变失能 | 否 |
| 是否导致永久惩罚 | 否 |

例子：荒野射手一次重伤，剩余恢复时间 3 天。在暴雨中触发伤员恶化后，当天不减少恢复时间，并额外 +1 天。于是剩余恢复时间变为 4 天。但该角色不会因此从一次重伤变成二次重伤。这类恶化提供压力，但不恶心玩家。

---

## 4.10 天气压力与重伤关系

第 6.2 中直接列出天气与重伤压力，不只写"承接第 5 类"。

| 天气 | 内部事件概率修正 | 与重伤关系 |
|---|---|---|
| 晴朗 | 0 | 不增加伤员压力 |
| 阴天 | 0 | 不增加伤员压力 |
| 小雨 | +0.05 | 潮湿环境轻微影响伤员 |
| 高温 | +0.05 | 缺水和烦躁轻微影响恢复 |
| 寒潮 | +0.05 | 取暖压力轻微影响恢复 |
| 浓雾 | +0.05 | 方向不安和心理压力增加 |
| 暴雨 | +0.10 | 潮湿、货物、伤员压力明显 |
| 酷热 | +0.10 | 缺水、烦躁、伤员恢复慢 |
| 严寒 | +0.10 | 取暖困难，治疗压力增加 |
| 重雾 | +0.10 | 恐惧、方向不安增加 |
| 灰烬风 | +0.15 | 污染焦虑和身体不适明显 |
| 灰烬风暴 | +0.30 | 极端压力，伤员最危险 |

6.2 中要强调：恶劣天气会提高伤员恶化和内部压力，但恶化只影响恢复天数，不升级重伤次数。若第 5 类天气最终表之后拆出更细的"伤员恶化概率"，以最终第 5 类天气表为准。

---

## 4.11 内伤、外伤、灰蚀、中毒

### 4.11.1 内伤与外伤

内伤和外伤只是战斗伤害判定类型，不是长期伤病系统。

| 类型 | 是否可被防御阻挡 | 是否有长期特殊效果 |
|---|---|---|
| 外伤 | 可以被护盾、护甲、屏障等阻挡或削减 | 无 |
| 内伤 | 不可阻挡，直接扣生命 | 无 |

内伤：不被护盾阻挡；不被护甲阻挡；不被屏障阻挡；没有额外长期伤病效果；只是一种不可阻挡伤害。

外伤：可以被护盾吸收；可以被护甲削减；可以被屏障吸收；没有额外长期伤病效果。

任何伤害类型，只要把生命降到 0 或以下，都触发重伤退出。

### 4.11.2 内伤来源

| 来源 | 说明 |
|---|---|
| 流血 | 触发时造成内伤 |
| 中毒 | 触发时造成内伤 |
| 过载 | 使用后造成内伤 |
| 灰蚀爆发 | 造成内伤 |
| 特殊敌人技能 | 可标注为内伤 |

### 4.11.3 外伤来源

| 来源 | 说明 |
|---|---|
| 普通攻击 | 默认外伤 |
| 野兽扑咬 | 默认外伤 |
| 强盗砍击 | 默认外伤 |
| 货车撞击类伤害 | 默认外伤 |
| 未特别标注的直接攻击 | 默认外伤 |

---

## 4.12 灰蚀爆发

灰蚀沿用第 6.1 的 3 层爆发规则，但爆发伤害类型明确改为内伤。

| 项目 | 规则 |
|---|---|
| 层数爆发 | 3 层时爆发 |
| 爆发伤害 | 8 点内伤 |
| 爆发后 | 清空灰蚀 |
| 是否可被护盾挡 | 否 |
| 是否可被护甲挡 | 否 |
| 是否可被屏障挡 | 否 |
| 是否可导致重伤 | 可以，若生命降到 0 或以下 |

灰蚀爆发造成内伤，更符合"灰烬侵蚀身体"的设定。

---

## 4.13 中毒

中毒是长期异常，和普通内伤不同。中毒造成的伤害属于内伤，但中毒层数可以在战斗结束后保留。

当前定稿：
> 战斗结束后，中毒层数减半，向上取整后保留。

| 战斗结束时中毒层数 | 战后保留中毒层数 |
|---|---:|---:|
| 0 | 0 |
| 1 | 1 |
| 2 | 1 |
| 3 | 2 |
| 4 | 2 |
| 5 | 3 |
| 6 | 3 |

中毒规则：

| 项目 | 规则 |
|---|---|
| 伤害类型 | 内伤 |
| 是否可被护盾挡 | 否 |
| 是否可被护甲挡 | 否 |
| 是否可被屏障挡 | 否 |
| 是否等于重伤 | 否 |
| 是否可导致重伤 | 可以，若中毒伤害把生命降到 0 或以下 |
| 战后保留 | 层数减半，向上取整 |
| 解除方式 | 城市治疗、药品、特殊事件，后续细化 |

中毒是需要长期管理的异常状态，但不应变成又一套复杂疾病系统。

---

## 4.14 不做轻伤

6.2 明确废弃轻伤。

| 内容 | 处理 |
|---|---|
| 轻伤 | 不做 |
| 轻伤惩罚 | 不做 |
| 轻伤恢复 | 不做 |
| 轻伤转重伤 | 不做 |

原因：轻伤会增加大量结算负担；重伤已经能提供足够惩罚；内伤/外伤已经覆盖战斗伤害类型；中毒已经承担长期异常压力；不需要再加一套轻伤状态。

正式口径：
> 《余烬商队》不做轻伤系统。队员要么还能战斗，要么生命降到 0 以下进入重伤退出。重伤是唯一伤病惩罚状态。

---

## 4.15 不做经验、不做熟练度、不做碎片

6.2 明确不采用普通 RPG 成长方式。

| 内容 | 是否采用 |
|---|---|
| 经验值 | 不采用 |
| 熟练度 | 不采用 |
| 技能碎片 | 不采用 |
| 记忆碎片 | 不采用 |
| 创伤记忆 | 不采用 |
| 重伤觉醒技能 | 不采用 |

原因：经验和熟练度太普通；碎片收集会让玩家烦；"获得半个记忆"没有好处；重伤后获得创伤记忆会污染重伤系统；游戏成长应该体现旧文明记忆和商队经营特色。

正式句：
> 要么获得完整记忆，要么不获得。没有记忆碎片，也没有技能碎片。

---

## 4.16 治疗经济接口（关联第 2.1 类经济系统）

### 4.16.1 自然休息回血

| 情况 | 效果 |
|---|---|
| 停下来休息 1 天 | 每名角色恢复 3 生命 |
| 有修补师随行并休息 1 天 | 每名角色恢复 5 生命 |
| 不休息 | 不自然回血 |

休息会消耗时间、推进订单期限、消耗补给。自然回血只解决轻伤，不能恢复重伤记录、最大生命惩罚或失能。

### 4.16.2 药包回血

| 情况 | 消耗 | 效果 |
|---|---|---|
| 普通角色使用药包 | 1 个药包 | 恢复 10 生命 |
| 有修补师使用药包 | 1 个药包 | 恢复 15 生命 |

药包规则：

| 项目 | 规则 |
|---|---|
| 基础价格 | 城市 50，城镇 70，驿站 75 |
| 战斗中使用 | 消耗 1 行动力 |
| 战斗外使用 | 消耗 1 天 |
| 是否处理重伤 | 不能 |
| 是否处理失能 | 不能 |

药包是路上保命资源，不是常规回血方式。安全点治疗明显比药包划算。

### 4.16.3 安全点普通治疗

安全点普通治疗一次恢复 20 生命。

| 地点/方式 | 价格 | 效果 | 单点成本 |
|---|---|---|---|
| 城市治疗 | 10 | 恢复 20 生命 | 0.5 |
| 城镇治疗 | 25 | 恢复 20 生命 | 1.25 |
| 驿站治疗 | 30 | 恢复 20 生命 | 1.5 |
| 药包，无修补师 | 50 | 恢复 10 生命 | 5 |
| 药包，有修补师 | 50 | 恢复 15 生命 | 3.33 |

到安全点治疗要明显更划算。玩家在路上用药包是为了活下来，不是为了省钱。

### 4.16.4 重伤与失能恢复价格

重伤/失能恢复**只在城市**处理。

| 状态 | 城市恢复价格 |
|---|---|
| 第一次重伤 | 200 |
| 第二次重伤 | 350 |
| 失能 | 500 |

可恢复地点限制：

| 方式 | 是否能恢复重伤/失能 |
|---|---|
| 城市 | 可以 |
| 城镇 | 不可以 |
| 驿站 | 不可以 |
| 药包 | 不可以 |
| 自然休息 | 不可以 |

防死档规则：正常游玩不做永久死亡和永久伤残；若整队失能，整队恢复总成本不超过 500；不按每人 500 叠加到 1500；玩家失误可以亏钱、亏货、亏时间、亏士气，但不能轻易死档。

### 4.16.5 地点治疗/恢复能力总表

| 服务 | 城市 | 城镇 | 驿站 |
|---|---|---|---|

| 重伤恢复 | ✅ | ❌ | ❌ |
| 失能恢复 | ✅ | ❌ | ❌ |

# 第五部分：记忆系统（6.2 全量）

---

## 5.1 记忆系统定位

"技"与"记忆"统一为一个系统，正式名称使用"记忆"。

记忆是旧文明遗留下来的技术、战斗协议、操作经验或意识模板。它可以让角色快速掌握某种强力攻击、防御、辅助或消耗能力。通过记忆释放出来的能力，通常比角色自身普通技能更强。

记忆不是：经验等级；熟练度奖励；技能碎片；创伤觉醒；受伤后领悟；普通技能书。

记忆是：旧文明技术；旧文明战斗协议；旧文明医疗方案；旧文明防御算法；旧文明火种操作方式；遗迹中的战术残响。

---

## 5.2 记忆分类

记忆分为四类：

| 类型 | 作用 |
|---|---|
| 攻击记忆 | 造成伤害 |
| 防御记忆 | 提供护盾、护甲、屏障或护车 |
| 辅助记忆 | 增伤、减伤、虚弱敌人、改变状态 |
| 消耗记忆 | 消耗特殊材料或特殊资源释放强效果 |

### 5.2.1 攻击记忆

| 记忆概念 | 效果方向 |
|---|---|
| 猎灰贯穿 | 对灰烬兽造成高额伤害 |
| 旧文明射击协议 | 提高射手攻击效果 |
| 灰烬处刑 | 对多异常状态目标造成高伤害 |

### 5.2.2 防御记忆

| 记忆概念 | 效果方向 |
|---|---|
| 旧文明护罩 | 提供屏障 |
| 车队守护协议 | 给货车护盾 |
| 反击盾阵 | 护盾 + 盾反强化 |

### 5.2.3 辅助记忆

| 记忆概念 | 效果方向 |
|---|---|
| 弱点解析 | 增加目标受到伤害 |
| 灰烬抑制 | 降低敌方伤害 |
| 旧文明诊断 | 减少中毒、流血、灰蚀压力 |

### 5.2.4 消耗记忆

消耗记忆需要消耗特殊材料或特殊资源释放强效果。

可消耗资源包括：火种；旧文明电池；高级零件；特殊弹药；车厢充能；稀有材料。

火种相关能力不单独命名为"火种技"，统一归入消耗记忆。

---

## 5.3 记忆位/技能位

之前"记忆槽只有 3 个"的说法废弃。因为第 6.1 中每名初始角色已经有 6 个基础技能，如果另设 3 个记忆槽，会和初始技能体系冲突。

当前定稿：
> 每名角色拥有若干出战技能/记忆位。初始技能占用这些位置。新获得的记忆需要替换已有技能或已有记忆，不能无限叠加。

| 阶段 | 每名角色出战技能/记忆位 |
|---|---:|
| 初始 | 6 |
| 中期成长 | 7 |
| 后期成长 | 8 |

说明：
- 初始三人各有 6 个基础技能，占满 6 个位置；
- 获得新记忆后，必须替换已有技能/记忆；
- 中后期可以通过成长方式增加 1–2 个位置；
- 不允许无限装记忆；
- 记忆和普通技能共享出战位。

例子：荒野射手初始拥有标记、放血箭、断筋箭、补射、会合射击、猎灰重箭 6 个技能。如果获得"猎灰贯穿记忆"，玩家必须决定替换哪个旧技能。到中期，如果射手获得第 7 个出战位，则可以多保留一个记忆或技能。

---

## 5.4 记忆获得

记忆完整获得，不存在碎片。

| 来源 | 获得方式 |
|---|---|
| 普通战斗 | 一般不掉记忆 |
| 精英战 | 小概率获得完整记忆 |
| Boss 战 | 较高概率获得强力记忆或火种相关奖励 |
| 旧文明遗迹 | 可能获得记忆 |
| 特殊订单 | 可获得指定记忆 |
| 城市研究/训练 | 可解锁基础记忆 |
| 特殊剧情 | 可获得专属记忆 |

正式句：记忆没有碎片。玩家不会收集"1/5 记忆碎片"。要么获得完整记忆，要么不获得。

---

## 5.5 记忆强化

记忆可以强化，但强化分为金币强化和火种强化。

### 5.5.1 金币强化

金币强化是小幅数值强化。

| 强化类型 | 例子 |
|---|---|
| 伤害 +1 | 攻击记忆小幅提高伤害 |
| 护盾 +1/+2 | 防御记忆小幅提高护盾 |
| 治疗 +1 | 辅助记忆小幅提高恢复 |
| 修车 +1 | 维护类记忆小幅提高修车 |
| 状态数值微调 | 小幅提高流血、中毒、迟滞相关效果 |

金币强化原则：便宜；稳定；小幅；不改变技能机制；不解锁高爆发；不大幅改变构筑。

### 5.5.2 火种强化

火种强化用于特殊强化，可以改变机制或进入高阶形态。

| 强化方向 | 例子 |
|---|---|
| 增加特殊效果 | 攻击附带状态 |
| 改变触发条件 | 对标记目标不消耗标记 |
| 强化连锁 | 击杀后额外抽牌 |
| 增加护车效果 | 防御记忆同时保护货车 |
| 解锁高条件爆发 | 对灰烬兽 Boss 破防窗口造成巨额伤害 |
| 转化为消耗记忆 | 消耗火种/旧文明电池释放强效果 |

火种强化原则：稀有；昂贵；强力；改变打法；与后期高条件爆发衔接。

---

## 5.6 记忆融合

记忆融合使用完整记忆，不使用碎片。

当前定稿：
> 记忆融合 = 用几个完整记忆，融合成一个更强的完整记忆。融合结果固定，不随机，不半随机。

| 项目 | 规则 |
|---|---|
| 材料 | 完整记忆 |
| 是否需要碎片 | 否 |
| 结果 | 固定产出指定记忆 |
| 是否随机 | 否 |
| 是否消耗材料 | 默认消耗 |
| 地点 | 城市、旧文明装置、特殊剧情点 |

例子：

| 融合材料 | 结果 |
|---|---|
| 放血类记忆 + 迟滞类记忆 | 撕裂箭记忆 |
| 防护类记忆 + 盾反类记忆 | 反击盾阵记忆 |
| 修车类记忆 + 护盾类记忆 | 复合补强记忆 |
| 猎灰类记忆 + 旧文明攻击协议 | 猎灰贯穿记忆 |
| 过载类记忆 + 火种电容 | 安全过载记忆 |

融合的乐趣来自取舍：玩家要不要消耗两个稳定记忆，换一个更强但更专门的记忆？

---

## 5.7 记忆融合经济接口（关联第 2.1 类经济系统）

### 5.7.1 金币消耗

记忆融合按参与记忆逐个计算。

| 参与记忆类型 | 每个记忆金币 |
|---|---|
| 同标签/同职业记忆 | 300 |
| 跨标签记忆 | 500 |

例子：护卫 + 护卫 = 600；射手 + 医师 = 1000；护卫 + 护卫 + 工匠 = 1100。

### 5.7.2 火种消耗

| 条件 | 火种消耗 |
|---|---|
| 全部同标签/同职业融合 | 0 |
| 只要跨标签融合 | +1 |
| 3 个记忆融合 | 额外 +1 |
| 第 4 个记忆 | 再 +2 |
| 第 5 个记忆 | 再 +3 |

设计目的：同类记忆融合可以相对容易；跨标签融合要更慎重；三个以上记忆融合需要火种限制；不让玩家无成本乱融合复合记忆。

### 5.7.3 火种词条价格

| 第几个词条 | 火种消耗 | 金币加工费 |
|---|---|---|
| 第 1 个词条 | 1 火种 | 300 |
| 第 2 个词条 | 2 火种 | 400 |
| 第 3 个词条 | 3 火种 | 500 |
| 第 4 个词条 | 4 火种 | 600 |

规则：火种词条不随机；每个技能有自己的可选词条池；玩家只能在已解锁、已适配词条里选择；多词条消耗递增；金币加工费是辅助成本，主要成本仍是火种。

# 第六部分：招募与商队扩展（6.2 全量）

---

## 6.1 招募系统

招募用于扩展商队成员，不是临时雇佣工具人。

### 6.1.1 招募类型

只保留三种：

| 类型 | 是否采用 | 说明 |
|---|---|---|
| 同职业变体 | 采用 | 护路人变体、射手变体、修补师变体 |
| 新职业 | 采用 | 新定位角色 |
| 新职业变体 | 采用 | 后期职业分支或稀有角色 |
| 临时雇员 | 不采用 | 不做临时护卫、临时向导、临时搬运工 |

不做临时雇员的原因：会削弱长期队伍感；容易变成工具人租赁；不利于记忆成长系统；和商队成员长期经营冲突。

### 6.1.2 招募节奏

招募不应在新手教程结束立刻开放第 4 人。

| 阶段 | 招募节奏 |
|---|---|
| 新手教程 | 不开放正式招募 |
| 早期 | 只出现候选人线索、剧情铺垫 |
| 中期 | 开始招募同职业变体/第 4 名队员 |
| 后期 | 招募新职业和新职业变体 |
| 大后期 | 可组织 5 人大型出行 |

招募需要条件，但不要做硬锁。可用条件包括：剧情推进；城市关系；商队声望；完成特定订单；救援事件；旧文明遗迹事件；金币成本；商队规模；货车空间；补给能力。

### 6.1.3 初始三人能力槽与后续变体方向（6.3 + 6.5）

6.5 处理初始职业变体与同职业招募方向。6.5 会在 6.3 的能力槽基础上扩展：

| 原职业 | 变体方向 |
|---|---|
| 护路人 | 护卫型 / 谈判型 |
| 荒野射手 | 信息型 / 伤害型 |
| 修补师 | 车辆型 / 治疗型 |

这些变体也应遵守：每名角色 4 个能力槽，2 个数值被动 + 2 个事件/世界接口。

---

## 6.2 商队规模不是升级等级

商队规模不作为硬性升级系统，不要求玩家点击"小型升级为中型"。

玩家可以在出发前选择本趟随行/上场人数。不同人数对应不同出行规模：

| 本趟出行人数 | 出行规模 | 每日补给消耗 | 战斗行动力 | 满员抽牌 |
|---|---:|---:|---:|---:|---:|
| 3 人 | 小型出行 | 1 | 3 | 7 |
| 4 人 | 中型出行 | 2 | 4 | 9 |
| 5 人 | 大型出行 | 3 | 5 | 11 |

系统不硬性禁止玩家带 4 人或 5 人，而是通过补给消耗、车厢空间、货物空间和路线距离形成软限制。

人数越多，战斗越强，但每日补给消耗越高。补给占用车厢格子，因此多带人会挤压货物空间，降低跑商收益。对于短途高危路线，玩家可能愿意多带人；对于长途普通订单，玩家可能更愿意少带人、多带货。

商队规模是对本趟出行配置和长期运营能力的分类，不是传统升级等级。

---

## 6.3 出行规模与商队评价

6.2 区分两个概念：

| 概念 | 含义 |
|---|---|
| 出行规模 | 本趟出发带几人，直接决定行动力、抽牌、补给消耗 |
| 商队评价 | 外界对玩家长期运营能力的分类，不是升级等级 |

### 6.3.1 出行规模

| 出行规模 | 判定 |
|---|---|
| 小型出行 | 本趟带 3 人 |
| 中型出行 | 本趟带 4 人 |
| 大型出行 | 本趟带 5 人 |

出行规模可以每趟变化。玩家即使已经有 6 名队员，也可以这趟只带 3 人跑短单。

### 6.3.2 商队评价

商队评价代表外界认为你"有多大本事"。

| 商队评价 | 说明 |
|---|---|
| 小型商队 | 通常只能稳定组织 3 人出行，适合短线小单 |
| 中型商队 | 已能稳定组织 4 人出行，能承担每日 2 补给和中等订单 |
| 大型商队 | 已能稳定组织 5 人出行，能承担每日 3 补给、高价值订单和长线风险 |

商队评价不需要玩家点升级。当玩家实际具备对应能力时，系统自然显示或由 NPC/商会评价变化体现。

---

## 6.4 允许临时降级出行

大型商队能力不是负担。玩家即使已经被视为大型商队，也可以选择小型出行。

| 玩家长期能力 | 本趟选择 | 本趟结算 |
|---|---|---|
| 已具备大型商队能力 | 只带 3 人跑短途 | 按小型出行结算：每日补给 1，行动力 3，抽 7 |
| 已具备大型商队能力 | 带 4 人跑中线 | 按中型出行结算：每日补给 2，行动力 4，抽 9 |
| 已具备大型商队能力 | 带 5 人跑高危长线 | 按大型出行结算：每日补给 3，行动力 5，抽 11 |

这样玩家不会觉得"我变成大型商队后，每天强制消耗 3 补给"，而是觉得"我的商队有大型出行能力，但这趟带多少人，由我自己决定"。

---

## 6.5 商队扩编的软限制

商队扩编不能做生硬硬锁。

不应写成：你没完成某任务，所以不能带第 4 人。

应该写成：玩家理论上可以扩编，但必须承担补给、空间、金币、招募、路线收益等实际压力。

软限制来源：

| 限制来源 | 表现 |
|---|---|
| 补给消耗 | 3 人每日 1，4 人每日 2，5 人每日 3 |
| 车厢格子 | 补给占格，多带人会挤压货物空间 |
| 重量 | 补给有重量，超重会形成软惩罚 |
| 金币成本 | 招募、治疗、强化、维护更贵 |
| 订单收益 | 低价值路线养不起大队伍 |
| 商队评价 | 强角色不愿加入太弱小商队 |
| 城市关系 | 部分角色需要城市关系或声望 |
| 路线压力 | 长线商路带多人消耗更高 |

当前不做的繁琐惩罚：

| 内容 | 当前处理 |
|---|---|
| 额外货车耐久损耗 | 暂不做 |
| 额外腐蚀 | 暂不做 |
| 人多导致内部事件额外上升 | 暂不做，后续可再看 |
| 敌人自动随人数增强 | 不在 6.2 强写，后续平衡处理 |
| 复杂车厢设施硬需求 | 暂不做 |

这样商队变大是有吸引力的，而不是一堆惩罚。

---

## 6.6 正常游玩不做永久惩罚

6.2 必须明确：
> 正常游玩中，不应因为普通战斗、随机事件、天气恶化而留下不可逆永久惩罚。

可以有：重伤；最大生命暂时下降；恢复天数；城市治疗成本；失能；订单时间损失；士气压力；补给压力。

但这些都应能通过合理方式恢复。

不能有：普通随机战斗永久断手；天气随机永久残废；第三次重伤永久删除角色；无法治疗的最大生命永久下降；正常玩法下永久废掉队员。

例外：

| 情况 | 是否可以有永久后果 |
|---|---|
| 明确剧情选择 | 可以 |
| 玩家主动接受代价的特殊事件 | 可以 |
| 终局剧情牺牲 | 可以 |
| 普通战斗随机伤害 | 不应有永久后果 |
| 普通天气恶化 | 不应有永久后果 |

这个原则可以保护玩家体验。

# 第七部分：补给与车厢系统（6.2 全量）

---

## 7.1 补给占格与软限制

已有资源口径中，补给属于车厢物资，需要占用车厢格子和重量。

当前 6.2 直接列出：

| 物资 | 占格 | 重量 | 是否堆叠 | 说明 |
|---|---|---|---:|---:|---|---|
| 普通补给 | 1 格 | 1 | 不堆叠 | 基础食物、水和生活消耗 |
| 压缩干粮 | 1 格 | 1 | 不堆叠 | 1 格可支撑 2 天 |
| 军用口粮 | 1 格 | 1 | 不堆叠 | 1 格可支撑 3 天 |
| 普通零件 | 1 格 | 2 | 不堆叠 | 修车与车体维护资源 |

### 7.1.1 车厢容量新口径

旧口径"每辆基础货车 6×7，总计 84 格"已废弃。
旧口径"初始总车厢 200 格"已废弃，200 格改为第一座城市升级后的容量目标，不是初始容量。

当前正式口径：

| 项目 | 口径 |
|---|---|
| 初始车队 | 2 辆基础货车 |
| 每辆基础货车 | 8×8 车厢格，共 64 格 |
| 初始总车厢空间 | 128 格 |
| 普通补给占格 | 1 格，重量 1，不堆叠 |
| 普通零件占格 | 1 格，重量 2，不堆叠 |

商路天数划分：

| 商路类型 | 大致天数 |
|---|---:|
| 短程商路 | 约 60 天 |
| 中程商路 | 约 100 天 |
| 长程商路 | 约 150 天 |
| 超长商路 | 约 200 天 |

初始车厢只要求支持短程商路，不要求支持中程、长程或超长程。玩家需要通过第一座城市的车厢升级、货车扩展、高级补给和更高效装载，逐步进入中程、长程和超长程商路。

车厢容量成长目标：

| 阶段 | 车厢容量 |
|---|---:|
| 初始基础车队 | 约 128 格 |
| 第一座城市升级后 | 约 200 格 |
| 中期扩展车队 | 约 320 格 |
| 后期大型车队 | 约 450 格 |
| 最终完全扩展 | 约 600 格 |

车厢限制原则：

| 限制 | 含义 |
|---|---|
| 空间 | 硬限制，格子塞不进去就不能带 |
| 重量 | 软限制，超重可以走，但会有惩罚 |
| 补给 | 占格、占重量，会挤压货物空间 |
| 订单货物 | 也占格，和补给竞争空间 |
| 战利品/灰烬材料 | 也占格，远行后期会进一步挤压空间 |

因此，多带人不是被系统禁止，而是会自然带来经营压力：人越多，每日补给消耗越高；补给越多，占用车厢格子越多；补给占格越多，货物空间越少；货物空间越少，跑商利润越低。

### 7.1.2 物资价格接口（关联第 2.1 类经济系统）

| 物资 | 城市 | 城镇 | 驿站 |
|---|---|---|---:|---:|
| 补给（每份） | 10 | 15 | 16 |
| 零件（每个） | 50 | 75 | 70 |
| 药包（每个） | 50 | 70 | 75 |

城镇/驿站库存限制：

| 地点 | 补给库存 | 零件库存 | 药包库存 |
|---|---|---|---:|---:|---:|
| 城市 | 大量/近似不限 | 大量/近似不限 | 大量/近似不限 |
| 城镇 | 20–40 | 3–6 | 5–10 |
| 驿站 | 15–30 | 5–8 | 3–6 |

### 7.1.3 商队耐久修理经济接口（关联第 2.1 类经济系统）

术语统一：修车 = 恢复商队耐久。

零件修理：

| 情况 | 消耗 | 效果 |
|---|---|---|
| 普通角色使用零件 | 1 个零件 | 恢复 10 点商队耐久 |
| 有修补师使用零件 | 1 个零件 | 恢复 15 点商队耐久 |

安全点修理服务（一次恢复 20 点商队耐久）：

| 地点/方式 | 价格 | 效果 | 单点成本 |
|---|---|---|---|
| 城市修理 | 10 | 恢复 20 商队耐久 | 0.5 |
| 驿站修理 | 25 | 恢复 20 商队耐久 | 1.25 |
| 城镇修理 | 30 | 恢复 20 商队耐久 | 1.5 |
| 零件，无修补师 | 50 | 恢复 10 商队耐久 | 5 |
| 零件，有修补师 | 50 | 恢复 15 商队耐久 | 3.33 |

设计目的：到城市就应该把商队耐久修好。零件贵在能带上路、能应急，不是省钱工具。

### 7.1.4 路线消耗模型（关联第 2.1 类经济系统）

```text
预计出行消耗 ≈ 天数 × 30 金币
```

拆分：补给 10/天 + 零件折算 5/天 + 药包折算 10/天 + 事件/修理预留 5/天。100 天路线预计消耗约 3000 金币。

单个订单收益 ≈ 预计消耗 × 1.4–1.7。长线真实收益约 20000–30000，规划好时更高。

## 7.2 人数、补给和路线距离示例

### 7.2.1 短程商路（约 60 天）

| 出行人数 | 每日补给 | 至少需要补给 | 占车厢格 |
|---|---:|---:|---:|---:|
| 3 人 | 1 | 60 | 60 格 |
| 4 人 | 2 | 120 | 120 格 |
| 5 人 | 3 | 180 | 180 格 |

初始车队 2 辆基础货车共 128 格。3 人跑 60 天短程商路需 60 格补给，剩余 68 格给订单货物、零件、药品、战利品和其他资源，空间较为充裕。4 人跑 60 天需 120 格补给，只剩 8 格，基本无法装货。5 人跑 60 天需 180 格，超过初始 128 格上限，无法实现。

短程高危路线中，3 人出行是初始车队的标准配置。如果玩家通过压缩干粮、军用口粮等高级补给降低占格，可以适当多带人。

### 7.2.2 中程商路（约 100 天）

| 出行人数 | 每日补给 | 至少需要补给 | 占车厢格 |
|---|---:|---:|---:|---:|
| 3 人 | 1 | 100 | 100 格 |
| 4 人 | 2 | 200 | 200 格 |
| 5 人 | 3 | 300 | 300 格 |

初始 128 格无法支撑 3 人跑 100 天中程商路（需 100 格补给，剩余 28 格不够装货）。玩家需要在第一座城市升级车厢到约 200 格后，才能用普通补给跑 3 人中程商路（200 格 - 100 格补给 = 100 格货物空间）。4 人中程需 200 格补给，需要中期扩展车队约 320 格才能实现。

### 7.2.3 长程商路（约 150 天）

| 出行人数 | 每日补给 | 至少需要补给 | 占车厢格 |
|---|---:|---:|---:|---:|
| 3 人 | 1 | 150 | 150 格 |
| 4 人 | 2 | 300 | 300 格 |
| 5 人 | 3 | 450 | 450 格 |

3 人长程需 150 格补给，需要中期扩展车队约 320 格才能跑（320 - 150 = 170 格货物空间）。5 人长程需 450 格，接近后期大型车队 450 格的全部容量，必须依靠高级补给（压缩干粮、军用口粮）降低占格才能实现。

### 7.2.4 超长商路（约 200 天）

| 出行人数 | 每日补给 | 至少需要补给 | 占车厢格 |
|---|---:|---:|---:|---:|
| 3 人 | 1 | 200 | 200 格 |
| 4 人 | 2 | 400 | 400 格 |
| 5 人 | 3 | 600 | 600 格 |

超长商路需要最终完全扩展约 600 格才能支撑 5 人出行。即使 3 人跑 200 天也需 200 格补给，需要后期大型车队 450 格以上才能保证货物空间。

玩家自然会感受到：我不是不能带 5 人，而是现在补给线不够、车厢空间不够、利润不够。如果硬带 5 人跑长线，我可能到不了下一个驿站，或者到达后根本赚不到钱。这就是软限制，而不是系统硬锁。

# 第八部分：能力槽系统（6.3 v0.2 全量）

---

## 8.1 能力槽总结构

每名角色基础拥有 4 个能力槽：

| 能力槽类型 | 数量 | 作用 |
|---|---:|---|
| 数值被动 | 2 个 | 后台固定生效，影响权重、信息、修车、资源、损失等 |
| 事件/世界接口 | 2 个 | 在特定事件中出现额外处理方式、提示或特殊选项 |
| 合计 | 4 个 | 形成角色在战斗外的专业性 |

基础格式：
```text
角色
├─ 数值被动 1
├─ 数值被动 2
├─ 事件/世界接口 1
└─ 事件/世界接口 2
```

---

## 8.2 数值被动规则

### 8.2.1 数值被动是什么

数值被动是角色的后台专业能力。它可以影响：事件结果权重；信息等级；资源收益权重；修车效率；损失权重；内部压力；车队安全；特定事件的风险分布。

例如：

| 角色 | 数值被动 | 作用 |
|---|---|---|
| 护路人 | 护卫 | 降低敌对、夜袭、混乱类事件最差结果权重 |
| 荒野射手 | 野外侦察 | 路线、敌情、痕迹信息等级 +1 |
| 修补师 | 修理知识 | 修车效率提高 |

### 8.2.2 数值被动固定生效

数值被动原则上**固定生效**，不再额外叠"是否触发"的概率。

原因：权重修正本身已经是概率影响。如果再叠一层"这个被动有没有触发"，结果会太飘，玩家也很难感知角色价值。

正确流程：
```text
事件进入结算
→ 检查随行角色
→ 如果角色具备相关数值被动
→ 直接修正对应权重/信息/数值
```

不采用以下流程：
```text
事件进入结算
→ 先判定角色被动是否触发
→ 触发后再修正权重
```

### 8.2.3 数值被动叠加规则

| 情况 | 规则 |
|---|---|
| 同一角色的两个数值被动 | 可以同时生效，但通常作用于不同对象 |
| 多名角色影响同一结果 | 后续可设上限，避免无限叠加 |
| 同类权重修正 | 暂时不做复杂叠乘，后续整合时统一平衡 |
| 初始三人阶段 | 以单角色单效果为主，避免多重叠加 |

建议临时上限：

| 修正对象 | 建议上限 |
|---|---|
| 降低坏结果权重 | 最多降低到原始权重的 60% |
| 提高好结果权重 | 最多提高到原始权重的 150% |
| 信息等级 | 最高不超过 5 |
| 初始无装备、无情报、无记忆 | 免费信息最高到 2 |

---

## 8.3 事件/世界接口规则

### 8.3.1 事件接口是什么

事件接口不是常驻主动技能。它只在满足条件的事件中出现。

它可以表现为：

| 接口形式 | 说明 |
|---|---|
| 新增选项 | 事件里多一个角色相关选择 |
| 标记选项 | 给已有选项加上"护路人建议/射手判断/修补师判断" |
| 替换选项 | 把一个普通选项替换成更专业的选项 |
| 风险提示 | 在选择前显示更准确信息 |
| 结果修正 | 选择某个选项后，降低坏结果或提高好结果 |
| 路线提示 | 提前看出风险、痕迹、结构问题 |
| 聚落提示 | 看出市场、治安、医疗、维修异常 |

### 8.3.2 事件接口不是主动技能

事件接口不做成常驻按钮。不应该出现：点击护路人技能：巡逻。正确表现是：当事件发生时，如果队伍中有对应角色，就在事件界面中出现相关提示或选项。

例子：
```text
事件：夜里货车旁有人影

普通选项：
1. 大声驱赶
2. 假装没看见
3. 花钱请守夜人

如果队伍中有护路人：
4. 【护路人】让护路人守在货车旁，先观察对方动向
```

### 8.3.3 事件接口出现条件

事件接口需要满足事件标签。

| 事件标签 | 可能触发接口角色 |
|---|---|
| 敌对/夜袭/威胁 | 护路人 |
| 护送/抢货/车队保护 | 护路人 |
| 路线/痕迹/伏击/野兽 | 荒野射手 |
| 资源/狩猎/采集 | 荒野射手 |
| 车损/道路障碍/结构故障 | 修补师 |
| 医疗/伤员/中毒/污染 | 修补师 |
| 聚落治安 | 护路人 |
| 聚落谣言 | 荒野射手 |
| 聚落维修 | 修补师 |
| 聚落医疗 | 修补师 |

### 8.3.4 事件接口强弱控制

事件接口按强弱分三档：

| 接口强度 | 出现方式 |
|---|---|
| 弱接口 | 可以固定出现 |
| 中接口 | 可以固定出现，但收益有限 |
| 强接口 | 需要条件限制，或只降低风险，不直接给高收益 |
| 剧情关键接口 | 不随机，按剧情/支线/主线条件出现 |

当前基础版建议：初始三人的事件接口以弱接口和中接口为主。不直接给大收益，不直接跳过核心风险。主要提供更稳的选择、更清楚的信息、更少的损失。

### 8.3.5 单个事件最多显示几个角色接口

为了避免事件界面太乱：

| 情况 | 规则 |
|---|---|
| 普通事件 | 最多显示 1 个角色接口 |
| 中大型事件 | 最多显示 2 个角色接口 |
| 主线/支线事件 | 可按剧情需要显示多个，后续再定 |

如果多个角色都符合条件，优先级按事件主题决定：

| 事件主题 | 优先角色 |
|---|---|
| 敌对威胁 | 护路人 |
| 路线痕迹 | 荒野射手 |
| 车损结构 | 修补师 |
| 医疗伤病 | 修补师 |
| 商队内部恐慌 | 护路人 |
| 聚落维修 | 修补师 |
| 聚落情报 | 荒野射手 |
| 聚落小治安 | 护路人 |

## 8.4 信息等级系统

### 8.4.1 信息等级 0–5

信息等级沿用旧版 6.3。

| 信息等级 | 名称 | 玩家看到什么 |
|---|---:|---|---|
| 0 | 无信息 | 不知道前方是什么 |
| 1 | 模糊信息 | 感觉前方有异常 |
| 2 | 类型信息 | 知道大概类型，比如野兽/强盗/道路问题 |
| 3 | 关键信息 | 知道主要风险点，比如伏击、车损、毒性、货损 |
| 4 | 详细信息 | 知道更明确的后果范围、敌人倾向、资源代价 |
| 5 | 精确信息 | 接近完整情报，可做最优准备 |

### 8.4.2 信息等级计算

```text
最终信息等级 =
事件基础信息
+ 区域信息
+ 角色数值被动
+ 角色事件接口
+ 工具/装备
+ 路线情报
+ 记忆/遗迹线索
```

最高不超过 5。

### 8.4.3 初始信息上限

初始三人无装备、无工具、无记忆、无额外情报时：

| 情况 | 上限 |
|---|---:|
| 普通隐藏/预警类事件 | 最高到 2 |
| 明显事件 | 可直接显示 2 |
| 需要专业工具的事件 | 初始角色不能直接到 3 |
| 高级遗迹/灰兽潮核心信息 | 初始角色不能直接看透 |

也就是说：初始角色可以让玩家"看出类型"，但不能免费"看穿全部"。

## 8.5 初始三人能力槽

### 8.5.1 护路人能力槽

| 槽位 | 类型 | 名称 | 核心作用 |
|---|---:|---|---|---|
| 1 | 数值被动 | 护卫 | 敌对、夜袭、护送、恐慌类事件中，最差混乱结果权重降低 |
| 2 | 数值被动 | 护车 | 货损、抢货、撤退中，订单货物、补给、零件损失权重降低 |
| 3 | 事件接口 | 挡在前面 | 在威胁、勒索、夜袭、护送事件中，提供更稳的正面处理方式 |
| 4 | 事件接口 | 维持秩序 | 在内部恐慌、聚落小治安、商队冲突事件中，降低局面失控风险 |

**护卫**适用范围：强盗事件；夜袭事件；护送事件；敌对人类事件；内部恐慌事件；聚落小治安事件。
效果：敌对、夜袭、护送、内部恐慌类事件中，最差混乱结果权重 ×0.8。
说明：护路人不能让危险消失，但能降低局面直接崩坏的概率。

**护车**适用范围：货损事件；抢货事件；撤退结算；车队保护；订单货物受损；补给/零件被夺。
效果：货损、抢货、撤退中，订单货物、补给、零件损失权重 ×0.85。
说明：护路人不是修车角色，但他懂得护住货车和货物。

**挡在前面**接口定位：护路人在外部威胁面前提供正面保护选项。

| 事件类型 | 接口表现 |
|---|---|
| 强盗勒索 | 增加"让护路人出面谈判/压阵"选项 |
| 夜袭 | 增加"护路人守住货车"选项 |
| 护送冲突 | 增加"护路人先挡住冲突"选项 |
| 聚落小治安 | 增加"护路人盯住货车附近的人"选项 |

效果倾向：降低货损（不一定避免战斗）；降低士气损失（商队更安心）；降低最差结果（但可能消耗时间/士气/少量资源）；提供稳妥方案（不是最高收益方案）。

**维持秩序**接口定位：护路人帮助商队在混乱、恐慌、争执中保持秩序。

| 事件类型 | 接口表现 |
|---|---|
| 内部争吵 | 增加"护路人压住局面"选项 |
| 士气恐慌 | 增加"护路人组织守夜/分工"选项 |
| 聚落骚乱 | 增加"护路人带队避开混乱区域"选项 |
| 商队冲突 | 增加"护路人划清边界"选项 |

效果倾向：降低士气损失（但不一定完全避免）；降低内部事件恶化（适合压力高时）；降低失控结果（不直接给资源）；增强队伍安全感（偏稳定，不偏收益）。

### 8.5.2 荒野射手能力槽

| 槽位 | 类型 | 名称 | 核心作用 |
|---|---:|---|---|---|
| 1 | 数值被动 | 野外侦察 | 路线、敌情、痕迹类事件信息等级 +1 |
| 2 | 数值被动 | 狩猎采集 | 自然资源事件中，补给类结果权重提高 |
| 3 | 事件接口 | 看懂痕迹 | 在路线、伏击、野兽、灰烬兽痕迹事件中，提供更准确信息或预警 |
| 4 | 事件接口 | 野外取食 | 在资源、狩猎、水源、采集事件中，提供更稳的资源获取方式 |

**野外侦察**适用范围：路线事件；敌情事件；痕迹事件；伏击预警；野兽活动；灰烬兽痕迹；雾天/林地/河谷等视野受限区域。
效果：路线、敌情、痕迹类事件的信息等级 +1。初始免费最高到 2。
说明：射手可以看出"这里不对劲"，但不能免费知道所有细节。

**狩猎采集**适用范围：自然资源事件；野外采集；水源事件；小型猎物；林地/河谷/残田牧野等资源区。
效果：自然资源事件中，补给类结果权重 ×1.2。
说明：荒野射手能提高找到食物和可用资源的机会，但不能凭空造补给。

**看懂痕迹**接口定位：射手通过脚印、车辙、兽迹、灰烬痕迹判断前方风险。

| 事件类型 | 接口表现 |
|---|---|
| 路线上有痕迹 | 增加"让射手判断痕迹" |
| 雾天前进 | 增加"射手先确认方向和动静" |
| 野兽活动 | 提示可能是普通野兽还是灰烬兽 |
| 伏击风险 | 提供更早预警 |
| 下雨后脚印/车辙 | 提供特殊判断信息 |

效果倾向：提高信息等级（常见效果）；降低伏击概率（但不完全免疫）；提前知道敌人类型（方便准备）；发现绕行机会（可能多耗时间或补给）。

**野外取食**接口定位：射手在野外资源事件中提供更专业的采集、狩猎和判断。

| 事件类型 | 接口表现 |
|---|---|
| 发现野果/菌类 | 判断是否安全 |
| 发现水源 | 判断是否可饮用 |
| 发现小型猎物 | 尝试狩猎 |
| 发现资源点 | 选择更稳的采集方式 |
| 暴雨后痕迹 | 判断是否值得追踪 |

效果倾向：提高补给收益（但可能耗时）；降低中毒/误采风险（尤其在湿地、林地）；提供保守采集选项（收益低但风险小）；提供冒险追踪选项（收益高但可能遇敌）。

### 8.5.3 修补师能力槽

| 槽位 | 类型 | 名称 | 核心作用 |
|---|---:|---|---|---|
| 1 | 数值被动 | 修理知识 | 修车效率提高；无零件临修效果提高 |
| 2 | 数值被动 | 现场诊断 | 结构、车损、普通设备故障信息等级 +1 |
| 3 | 事件接口 | 临场修补 | 在车损、道路障碍、维修事件中，提供临时处理方式 |
| 4 | 事件接口 | 伤病判断 | 在伤员、中毒、污染、聚落医疗事件中，提供更准确判断或低风险处理 |

**修理知识**适用范围：车队维修；临时修车；车损事件；零件不足时的应急处理。
效果：1 零件修车 +16。无零件临时修车 +5。
说明：修补师是当前队伍里最直接影响车况恢复的角色。

**现场诊断**适用范围：结构问题；车损问题；道路障碍；普通设备故障；旧文明普通装置；聚落维修服务异常。
效果：结构、车损、普通设备故障类事件信息等级 +1。初始免费最高到 2。
说明：修补师能判断"大概坏在哪里"，但不能免费破解高级旧文明设备。

**临场修补**接口定位：修补师在车损、道路障碍、维修服务波动中提供临时处理方式。

| 事件类型 | 接口表现 |
|---|---|
| 道路损坏 | 增加"让修补师检查车轴/车轮" |
| 车体受损 | 增加"临时加固车体" |
| 没有零件 | 增加"用废料临时补一下" |
| 聚落修车涨价 | 增加"修补师判断是否值得修" |
| 维修排队 | 增加"修补师先做基础处理" |

效果倾向：降低车损（不一定完全修好）；降低维修费用（可能只适用于小修）；降低修车耗时（不替代城市工坊）；提供临时方案（之后仍可能需要正式维修）。

**伤病判断**接口定位：修补师在伤员、毒性、污染、医疗服务事件中提供更准确判断。

| 事件类型 | 接口表现 |
|---|---|
| 队员受伤 | 增加"修补师先判断伤势" |
| 中毒风险 | 提示毒性来源或处理方式 |
| 灰烬污染 | 判断是否需要立刻处理 |
| 聚落医疗涨价 | 判断是否必须现在治疗 |
| 医馆排队 | 提供临时护理方案 |

效果倾向：降低伤情恶化（但不免费治好）；降低误治风险（尤其是中毒/污染）；提供是否等待的判断（适合聚落医疗波动）；降低轻伤处理成本（但重伤仍需城市治疗）。

## 9.3 记忆可以产生主动技和被动技

记忆不是单纯材料。记忆可以通过融合、设施处理或火种引导，转化为角色可用的技艺。

```text
记忆技艺
├─ 主动技
└─ 被动技
```

主动技：

| 类型 | 说明 |
|---|---|
| 攻击技 | 造成伤害，例如射箭、重击、放血箭 |
| 防御技 | 获得护盾、护车、防护，例如横身拦截、车队防护 |
| 辅助技 | 治疗、净化、标记、修车、加固等 |

被动技：

| 类型 | 说明 |
|---|---|
| 战斗被动 | 战斗中自动生效 |
| 远行被动 | 路线、天气、野外资源相关 |
| 行商被动 | 交易、议价、市场、订单相关 |
| 事件被动 | 野外事件、聚落事件、内部事件中生效 |

注意：
```text
主动/被动是技艺表现形式。
职业标签是记忆、设施、火种是否适配的规则。
两者不是一回事。
```

## 9.4 技能成长两条线

技能成长分为两条互相独立的线：

```text
技能成长
├─ 普通升级：金币 + 设施，用于数值升级/状态升级
└─ 火种升级：火种 + 特殊设施，用于词条/后缀/特殊效果
```

关键规则：

| 项目 | 规则 |
|---|---|
| 普通升级 | 提升技能内部的数值项或状态项 |
| 火种升级 | 给技能添加词条/后缀/特殊效果 |
| 是否互相替代 | 不互相替代 |
| 高普通等级是否等于有火种词条 | 不等于 |
| 火种词条多是否等于普通等级高 | 不等于 |

例：一个普通等级很高的射箭，可以没有任何火种词条。一个普通等级较低的射箭，也可以拥有火种词条。

---

## 9.5 普通升级：数值升级与状态升级

普通升级不再只是"整张技能卡自动升一级"。每次普通升级时，玩家可以在该技能允许的升级项中选择。

普通升级分为两类：
```text
普通升级
├─ 数值升级
└─ 状态升级
```

### 9.5.1 数值升级

数值升级负责直接增加数值。

| 类型 | 示例 |
|---|---|
| 伤害数值 | 伤害 +1 |
| 护盾数值 | 护盾 +1 |
| 治疗数值 | 治疗 +1 |
| 修车数值 | 修车 +1 |
| 车体防护 | 车体防护 +1 |

数值升级相对基础，价格应低于状态升级，但具体价格本版不定。

### 9.5.2 状态升级

状态升级负责让技能施加的正面/负面状态更强。

| 类型 | 示例 |
|---|---|
| 负面状态层数 | 标记层数 +1 |
| 负面状态层数 | 流血层数 +1 |
| 负面状态层数 | 中毒层数 +1 |
| 正面状态强度 | 护盾强度 +1 |
| 正面状态层数 | 防护层数 +1 |
| 净化处理 | 净化层数 +1 |

注意：不写"强化标记"。要写成具体规则，例如"标记层数 +1"。状态升级比数值升级更贵，但具体价格后续再定。

---

## 9.6 普通等级显示方式

普通等级可以作为 UI 上的总显示等级，但内部应拆成：
```text
普通总等级 = 数值升级点 + 状态升级点
```

例如：精确射击 Lv6 = 数值升级点 5 + 状态升级点 1。这样玩家看到的是一个清楚的等级，系统内部也能区分"伤害成长"和"状态成长"。

## 9.7 技能融合规则

技能融合不设主技能/副技能。多个技能或记忆可以融合成新技能/复合记忆。

融合时分别继承：
```text
新技能数值升级点 = 参与融合技能中最高的数值升级点
新技能状态升级点 = 参与融合技能中最高的状态升级点
新技能普通总等级 = 新技能数值升级点 + 新技能状态升级点
```

融合后，新技能允许玩家在新技能可用升级项中重新分配继承来的升级点。

### 9.7.1 融合示例：射击 + 标记 = 精确射击

假设：
```text
射击
- 数值升级点：5
- 主要效果：造成伤害

标记
- 基础效果：施加 1 层标记
- 状态升级点：1
- 当前效果：施加 2 层标记
```

融合：射击 + 标记 → 精确射击

继承：
```text
精确射击
- 数值升级点：5
- 状态升级点：1
- 普通总等级：6
```

效果可以表现为：造成伤害，并施加 2 层标记。

| 来源 | 继承内容 |
|---|---|
| 射击 | 继承 5 点数值升级，可用于提高伤害 |
| 标记 | 继承 1 点状态升级，使标记从 1 层变成 2 层 |

这条规则的目的：高等级伤害技能不会因为融合被清零；已经升级过的状态技能也不会浪费。

---

## 9.8 火种升级与词条池

火种升级独立于普通等级。

火种负责：

| 作用 | 示例 |
|---|---|
| 添加词条 | 射箭【重击】 |
| 添加特殊效果 | 射箭【穿甲】 |
| 改变方向 | 治疗箭、车厢盾架 |
| 解锁特殊处理 | 治疗【净毒】、车体修理【加固】 |

火种词条不随机，也不是所有技能都能随便加。

采用规则：
```text
每个技能有自己的火种词条池。
玩家在已解锁、且该技能允许的词条池中自由选择。
```

火种词条可以多个，但成本递增。

| 第几个火种词条 | 消耗倾向 |
|---|---:|---|
| 第 1 个 | 低 |
| 第 2 个 | 更高 |
| 第 3 个 | 很高 |
| 后续 | 继续递增，具体公式后续再定 |

---

## 9.9 火种词条类型限制

火种词条需要类型限制，避免不合理组合。

| 词条类型 | 可用于 |
|---|---|
| 攻击词条 | 攻击技，例如射箭、重击、放血箭 |
| 防御词条 | 防御技，例如横身拦截、稳住阵线 |
| 治疗词条 | 治疗技，例如简易治疗 |
| 修补词条 | 修车/加固类技能 |
| 状态词条 | 标记、流血、中毒、净化等相关技能 |
| 行商词条 | 后续行商技能 |
| 灰烬词条 | 灰烬净化、污染、火种相关技能 |

规则：治疗类技能不能随便加【重击】。攻击技、防御技、治疗技、修补技、状态技等，各自拥有自己的可选词条池。

---

## 9.10 融合时火种词条处理

火种词条很稀有，融合时原则上尽量保留。但必须经过新技能词条池检查。

| 情况 | 处理 |
|---|---|
| 新技能支持该词条 | 保留 |
| 新技能不支持该词条 | 不强行保留，可转为重选或暂存 |
| 多个融合材料都有火种词条 | 所有适配词条尽量保留，成本/规则后续平衡 |
| 词条类型明显不适配 | 不能硬塞进新技能 |

例：
```text
射箭【穿甲】 + 标记 → 精确射击
如果精确射击属于攻击技，并支持【穿甲】，则该词条可以保留。

简易治疗 + 重击类词条
如果新技能是纯治疗技，则不能强行保留攻击词条。
```

### 9.12.1 技能升级价格（关联第 2.1 类经济系统）

注意：技能成长主要依赖城镇/城市设施，不做野外随地升级。

**数值升级**（伤害+1、护盾+1、治疗+1、修车+1 等）：

公式：第 N 次数值升级价格 = 300 + 20 × (N - 1)

| 第几次 | 价格 |
|---|---:|---:|
| 第 1 次 | 300 |
| 第 2 次 | 320 |
| 第 3 次 | 340 |
| 第 10 次 | 480 |
| 第 20 次 | 680 |

不用乘法增长，避免后期升不动。

**状态升级**（标记层数+1、流血层数+1、正面状态层数/强度+1 等）：

公式：第 N 次状态升级价格 = 1000 + 200 × (N - 1)

| 第几次 | 价格 |
|---|---:|---:|
| 第 1 次 | 1000 |
| 第 2 次 | 1200 |
| 第 3 次 | 1400 |
| 第 5 次 | 1800 |
| 第 10 次 | 2800 |

状态升级比数值升级贵，因为状态升级通常强于单纯 +1 数值。例如流血层数 +1 可能带来多次行动触发的额外伤害，价值高于一次普通伤害 +1。

## 10.1 本章核心结论

装备系统不做纯数值装备，也不做每件装备都复杂独特的纯机制装备。

当前原则是：
```text
装备 = 基础数值 + 清楚机制 + 职业限制 + 使用取舍 + 可成长空间
```

本章核心口径：
1. 装备系统分为四大类：角色装备、商队模块、道具系统、特殊装备。
2. 角色装备包括：武器、防具、工具、挂饰。
3. 商队模块是整个商队共用，不绑定单辆车。
4. 商队耐久、资源、订单货损统一结算；不做单车独立耐久，不做单车独立装备树。
5. 基础商队模块栏位为：装甲槽 ×1、车载武器槽 ×1、照明/警戒槽 ×2、环境槽 ×1、车体结构槽 ×2、商队凭证槽 ×1、战术道具槽 ×3。
6. 同类型/同子类模块互斥，不能同时装备两个完全相同或互斥同类模块。
7. 储存容器不占模块槽，只占商队容量。
8. 灰烬封存箱是储存容器，不是特殊装备。
9. 灰烬是常见但危险的环境/材料风险，不应被神话化。
10. 旧文明装备可以残件使用，效果弱；可以逐步修复或启动，恢复更强效果。
11. 普通装备可以多次强化，但本章不写强化曲线；普通装备强化不用火种。
12. 装备不做耐久、不做随机词条、不做传统品质等级。
13. 遗迹探索嵌入跑订单过程，玩家可以边跑订单边探索世界。

## 10.4 灰烬、旧文明与装备的关系

### 10.4.1 灰烬不要神话化

灰烬不是神话资源，也不是一沾上就进入最高级装备系统的东西。
```text
灰烬是常见但危险的环境/材料/污染风险。
它和受潮、寒湿、污染扩散属于一类生存压力，只是严重程度更高。
```

因此：

| 不要这样写 | 应该这样写 |
|---|---|
| 涉及灰烬就很神秘 | 灰烬是常见风险，只是更危险 |
| 灰烬封存箱算特殊装备 | 灰烬封存箱只是储存容器 |
| 灰烬物品都要火种 | 普通灰烬材料不一定需要火种 |
| 灰烬 = 旧文明 | 灰烬和旧文明可能有关，但不是同一概念 |

### 10.4.2 旧文明装备才是特殊装备

旧文明装备特点：

| 特点 | 说明 |
|---|---|
| 残件可用 | 残件状态下可使用，但效果弱 |
| 可逐步修复 | 修复后恢复更强效果 |
| 不走普通强化 | 需要旧文明材料、修复方案、专门工匠或设施 |
| 部分需要火种启动 | 强力旧文明装备可消耗火种或特殊能量 |
| 代价明显 | 修复成本高、使用有限制、维护困难 |

例子：旧文明雾灯。

| 阶段 | 效果 |
|---|---|
| 残件状态 | 可用，但只提供弱路线判断 |
| 一阶修复 | 重雾路线中更稳定识别道路 |
| 二阶修复 | 可降低重雾中的伏击和迷路风险 |
| 完全修复 | 效果强，但需要维护材料或特殊滤芯 |

## 10.7 角色装备大类

### 10.7.1 武器类

| 武器大类 | 小类 | 主要标签 | 功能方向 |
|---|---|---|---|
| 盾类 | 小盾、轻盾、重盾 | 护卫 | 防御、护盾、保护商队 |
| 弓类 | 短弓、长弓、猎弓 | 射手/猎手 | 标记、远程、稳定输出 |
| 弩类 | 轻弩、重弩、机关弩 | 射手/工匠 | 高伤、装填、爆发 |
| 近战类 | 短剑、斧、长柄武器 | 护卫/行商 | 拦截、反击、稳定伤害 |
| 工具武器 | 修补锤、撬棍、拆解钳 | 工匠 | 修理、拆解、辅助攻击 |

普通武器初始加成一般不超过 10 点，单件装备初始总数值收益一般不超过 20 点。目的不是让新武器完全碾压旧武器，而是让玩家选择更适合构筑的装备。

**重盾示例**：双手防御武器，护卫标签，占双手。基础数值：防御类技能提供的护盾 +3。机制：使用防御类技能时，可额外给商队提供少量护盾或降低本回合商队受击压力。限制：不能装备弓、弩、双手攻击武器；攻击类技能伤害 -2。

**猎弓示例**：双手攻击武器，射手/猎手标签，占双手。基础数值：射击伤害 +2。机制：攻击被标记目标时，额外 +2 伤害。限制：对高护甲敌人收益较低。

**重弩示例**：双手攻击武器，射手/工匠标签，占双手。基础数值：射击伤害 +5。机制：每场战斗第一次射击类技能额外造成伤害。限制：重弩强化每回合最多触发 1 次，或使用后进入装填限制。

### 10.7.2 防具类

防具系统接入已有防御三层。

| 防御层 | 定位 | 装备接口 |
|---|---|---|
| 护盾 | 临时防护，战斗内常见 | 轻甲、盾牌、战术道具可提供 |
| 护甲 | 更稳定的长期防护层 | 防具和车辆装甲最适合提供 |
| 屏障 | 高级防护，稀有强力 | 旧文明装备、特殊装备少量提供 |

防具可以提供：开局护盾、开局护甲、减伤、受潮抵抗、寒湿抵抗、灰蚀抵抗、特定地形/天气惩罚降低。

防具不要写：降低重伤风险、防止重伤、生命归零不重伤、无限续命。生命归零进入重伤是已有规则，装备不能随意破坏。

| 防具大类 | 小类 | 作用 |
|---|---|---|
| 头部 | 头盔、护目镜、面罩 | 护甲、雾中观察、灰烬防护 |
| 躯干 | 轻甲、中甲、重甲、防护服、防潮披风 | 护甲、护盾、天气抵抗 |
| 手部 | 手套、护手、工具手套 | 工具效率、受伤保护 |
| 腿部 | 护腿、绑腿 | 行路事件、迟缓抵抗 |
| 足部 | 靴子、泥地靴、防寒靴 | 地形、天气、行路风险 |

### 10.7.3 工具类

工具不要用"挎包""腰包"这类像单纯加容量的名字，改成具体器具。

| 工具 | 标签 | 功能方向 |
|---|---|---|
| 包扎器具组 | 医师 | 提高治疗技能或药包效果 |
| 夹板固定组 | 医师 | 战斗外处理伤势，提高休息回血 |
| 维修工具箱 | 工匠 | 提高零件、修理、临时修复效果 |
| 拆解工具组 | 工匠 | 对护甲、机械、旧文明装置有额外处理 |
| 侦察镜 | 猎手/行商 | 获得免费粗略传闻 |
| 测灰器 | 灰烬/工匠 | 判断灰烬污染、灰烬材料风险 |
| 交易凭牌 | 行商 | 交易、聚落交涉、价格稳定 |

### 10.7.4 挂饰类

挂饰不要太强，主要做轻量被动和事件接口。

| 挂饰 | 方向 |

| 护符 | 士气、灰烬污染、特殊事件 |
| 小型凭证 | 聚落交涉、驿站服务 |
| 旧世铭牌 | 遗迹识别、旧文明设备互动 |
| 信物 | 特定城市、订单、势力相关加成 |

## 10.8 商队模块系统

### 10.8.1 核心结算口径

多辆车不做独立装备系统。车辆装备统一改为商队级模块系统。

```text
所有车辆共同组成商队。
商队耐久统一结算。
资源统一结算。
订单货损统一结算。
车辆装备作用于整个商队。
不做每辆车独立耐久。
不做每辆车独立装备树。
```

如果每辆车独立装备，会产生明显问题：A 车装护甲为什么保护 B 车、B 车装宽轮为什么 A 车也不坏、C 车装防潮箱到底保护哪辆车的货。这些都会迫使系统做单车结算，复杂度太高。

因此本章采用商队整体装备系统。多辆车只影响商队容量、商队耐久上限、模块槽数量、重量和维护成本。

### 10.8.2 基础商队模块栏位总表

| 槽位 | 数量 | 可装内容 | 规则 |
|---|---:|---|---|
| 装甲槽 | 1 | 装甲板、防撞板、护货挡板 | 只能选 1 个 |
| 车载武器槽 | 1 | 车载弩炮、车载弩、车载投射器 | 只能选 1 个，通常额外占容量 |
| 照明/警戒槽 | 2 | 雾灯、警戒铃、照明器、警戒旗、探照灯 | 同类型互斥，不能装两个完全相同或同子类互斥装备 |
| 环境槽 | 1 | 防潮板、遮阳棚、防寒罩、灰烬滤网 | 只能选 1 个 |
| 车体结构槽 | 2 | 车轮、车架、减震、牵引、制动类改装 | 可装 2 个不同子类，同类型互斥 |
| 商队凭证槽 | 1 | 商牌、路引、通行凭证、势力证明 | 只能选 1 个 |
| 战术道具槽 | 3 | 绊索器、信号枪、便携盾架、探路杆组 | 不一定占容量，但必须占槽位 |

### 10.8.3 同类型互斥规则

商队模块采用"同类型互斥"规则。
```text
两个同类型/同子类的模块不能同时装备。
不能装备两个完全相同模块。
不能通过重复装备同一类模块叠加同类效果。
```

例子：

| 搭配 | 是否允许 | 原因 |
|---|---|---|
| 宽轮 + 加固车架 | 允许 | 车轮类 + 车架类，不同子类 |
| 宽轮 + 减震结构 | 允许 | 车轮类 + 减震类，不同子类 |
| 加固车架 + 强化牵引架 | 允许 | 车架类 + 牵引类，不同子类 |
| 宽轮 + 泥地轮 | 不允许 | 都是车轮类，互斥 |
| 减震结构 + 软轴结构 | 不允许 | 都是减震类，互斥 |
| 加固车架 + 加固车架 | 不允许 | 完全相同 |
| 雾灯 + 警戒铃 | 允许 | 照明/警戒槽中不同子类 |
| 雾灯 + 旧文明雾灯 | 不允许或需特殊替换规则 | 同属雾灯类，互斥 |
| 警戒铃 + 警戒铃 | 不允许 | 完全相同 |

### 10.8.4 装甲槽 ×1

装甲槽用于商队防御。

| 装备 | 效果方向 |
|---|---|
| 装甲板 | 战斗开始给商队护甲 |
| 防撞板 | 路况事件中减少商队耐久损失 |
| 护货挡板 | 降低货损 |

装甲槽只能选一个，玩家要在抗战斗、抗路况、保货物之间取舍。

### 10.8.5 车载武器槽 ×1

车载武器槽用于车队战斗火力。

| 装备 | 效果方向 |
|---|---|
| 车载弩炮 | 每场战斗提供一次车载射击 |
| 轻型车载弩 | 伤害较低，但占用小、弹药便宜 |
| 重型车载弩炮 | 伤害高，占容量大，弹药贵 |
| 车载投射器 | 发射燃烧瓶、爆裂弹等特殊弹药 |

车载武器通常额外占商队容量，部分还需要弹药。装车载武器等于少装货，换战斗火力。

### 10.8.6 照明/警戒槽 ×2

照明/警戒槽用于侦察、夜间、重雾、遗迹和伏击预警。

| 装备 | 作用 |
|---|---|
| 雾灯 | 重雾中改善路线判断，降低伏击/迷路压力 |
| 照明器 | 夜间、遗迹、灰暗区域探索辅助 |
| 警戒铃 | 降低夜间偷袭、偷货、伏击压力 |
| 警戒旗 | 更容易被友方/商队识别，降低误判冲突 |
| 探照灯 | 遗迹探索、夜间事件中提供更强信息 |

照明/警戒槽有 2 个，但同类型互斥。

### 10.8.7 环境槽 ×1

环境槽用于应对天气、灰烬、寒冷、炎热和潮湿。

| 装备 | 效果方向 |
|---|---|
| 防潮板 | 降低雨天受潮风险 |
| 遮阳棚 | 酷热天气降低补给压力 |
| 防寒罩 | 严寒天气降低恢复惩罚 |
| 灰烬滤网 | 灰烬风/灰烬污染区域降低污染扩散 |

环境槽只有 1 个，玩家要根据路线情报选择环境适配方向。

### 10.8.8 车体结构槽 ×2

车体结构槽用于商队行路、稳定性、车架和道路适配。

| 子类 | 例子 | 作用 |
|---|---|---|
| 车轮类 | 宽轮、泥地轮、铁箍轮 | 应对泥地、湿地、雨后道路 |
| 车架类 | 加固车架、轻量车架 | 改变商队耐久上限或重量 |
| 减震类 | 减震结构、软轴结构 | 应对山地、烂路、遗迹道路 |
| 牵引类 | 强化牵引架、双挂牵引 | 降低绕路、坡地、拖拽压力 |
| 制动类 | 加强制动、坡道制动器 | 降低下坡、湿滑道路风险 |

车体结构槽有 2 个，可以装两个不同子类；同类型/同子类互斥。

### 10.8.9 商队凭证槽 ×1

商队凭证槽吸收原"车队牌"的部分功能。

| 凭证 | 效果方向 |
|---|---|
| 商牌 | 某些城市/城镇交易更稳定 |
| 路引 | 特定路线通行、事件交涉 |
| 驿站通行凭证 | 驿站服务或情报更方便 |
| 势力证明 | 特定订单、势力事件有额外选项 |

基础商队只能挂 1 个凭证，玩家要选择这趟路线使用哪种身份或凭证。

### 10.8.10 战术道具槽 ×3

战术道具可以不占商队容量，但必须占槽，不能无限携带。

| 战术道具 | 使用限制 |
|---|---|
| 绊索器 | 每场战斗 1 次 |
| 信号枪 | 每条路线段 1 次，需要信号弹 |
| 便携盾架 | 每场战斗 1 次 |
| 探路杆组 | 路况事件中每路线段 1 次 |
| 警戒哨 | 驻留/夜间事件中触发 |

## 10.9 储存容器

储存容器不占模块槽，只占商队容量。

| 储存容器 | 占用 |
|---|---|
| 防潮箱 | 商队容量 |
| 灰烬封存箱 | 商队容量 |
| 药品箱 | 商队容量 |
| 材料箱 | 商队容量 |
| 特殊货物箱 | 商队容量 |

例子：防潮箱：占 2 点商队容量，可保护一定数量怕潮货物。灰烬封存箱：占 2 点商队容量，可封存灰烬材料或污染货物。

储存容器不是模块槽装备，不和装甲槽、环境槽等冲突。它的代价是占用商队容量，减少可装订单货物或资源。

### 10.9.1 商队模块例子

| 模块 | 分类 | 占用 | 主要效果 | 限制 |
|---|---|---|---|---|---|
| 装甲板 | 装甲装备 | 装甲槽 | 战斗开始时商队获得 10 点商队护甲 | 增加重量，占唯一装甲槽 |
| 宽轮 | 车体结构/车轮类 | 车体结构槽 | 雨后泥地、湿地、烂路事件中降低商队耐久损耗 | 与其他车轮类互斥 |
| 防潮板 | 环境装备 | 环境槽 | 雨天货物受潮风险降低 | 只对雨天/潮湿事件有效 |
| 防潮箱 | 储存容器 | 商队容量 | 箱内怕潮货物不受雨天受潮影响 | 只保护箱内容量 |
| 灰烬封存箱 | 储存容器 | 商队容量 | 封存灰烬材料或污染货物，降低污染扩散风险 | 不保护普通货物，不算特殊装备 |
| 车载弩炮 | 车载武器 | 车载武器槽 + 容量 | 每场战斗发动 1 次车载射击 | 占容量，伏击首回合不可用，可需要弹药 |

# 第十一部分：道具系统（6.7 全量）

## 11.1 道具分类

道具系统分三类。

| 类型 | 定义 | 例子 |
|---|---|---|
| 战术道具 | 不直接消失，但有次数、冷却、装填、弹药或路线段限制 | 绊索器、信号枪、便携盾架 |
| 消耗道具 | 用一次少一个，或使用后消失 | 药包、燃烧瓶、爆裂箭、烟雾罐、净灰瓶 |
| 多次使用工具 | 可重复使用，通常用于事件或路线 | 遮雨布、绳索、探路杆 |

修正口径：
```text
烟雾罐明显是一次性消耗品，不归战术道具。
遮雨布明显可重复使用，不归一次性消耗品。
不做"防止重伤"道具。
生命归零仍按重伤规则处理。
```

### 11.1.1 战术道具大类

| 类型 | 作用 |
|---|---|
| 控制类 | 迟缓、限制敌人、打断行动 |
| 支援类 | 帮助撤退、增加治疗量、提供临时护盾 |
| 侦察类 | 降低伏击风险、获得免费传闻 |
| 车队类 | 保护商队耐久、保护货物 |
| 环境类 | 应对天气、地形、灰烬污染 |

### 11.1.2 战术道具例子

| 道具 | 类型 | 限制 | 效果 |
|---|---|---|---|
| 绊索器 | 战术道具 | 每场战斗 1 次 | 对一个地面敌人施加迟缓 |
| 信号枪 | 战术道具 | 每条路线段 1 次，需要信号弹 | 降低下一次伏击风险，或在撤退事件中增加撤退进度 |
| 便携盾架 | 战术道具 | 每场战斗 1 次 | 给商队或一名角色提供临时护盾，不能防止重伤 |

### 11.1.3 消耗道具例子

| 道具 | 类型 | 效果 | 限制 |
|---|---|---|---|
| 烟雾罐 | 消耗道具 | 本回合降低敌方远程伤害，伏击战中可增加撤退窗口 | 使用后消耗 1 个，雨天/灰烬风中效果降低 |
| 燃烧瓶 | 消耗道具 | 造成火焰伤害，并施加灼烧 | 雨天火焰伤害按天气规则降低 |
| 爆裂箭 | 消耗道具 | 射手技能可消耗爆裂箭造成额外伤害或小范围伤害 | 需要弓/弩类装备，无射手标签不能完整使用 |

# 第十二部分：特殊装备与旧文明装备（6.7 全量）

---

## 12.1 特殊装备定位

特殊装备主要指：

| 类型 | 说明 |
|---|---|
| 旧文明装备 | 遗迹、废墟、生产基地中获得的残件或设备 |
| 火种启动装备 | 需要火种或特殊能量启动的强力装备 |
| 遗迹修复装备 | 需要修复方案、旧文明材料、专门工匠修复 |

特殊装备可以是角色装备、商队模块，也可以是道具。

---

## 12.2 旧文明装备规则

| 规则 | 内容 |
|---|---|
| 残件可用 | 残件状态下可使用，但效果弱 |
| 可逐步修复 | 通过材料、图纸、专门工匠恢复效果 |
| 不走普通强化 | 不用普通金币材料强化 |
| 修复代价高 | 需要旧文明残件、修复方案、特殊材料 |
| 收益明显 | 完成修复后效果要足够强 |
| 部分可火种启动 | 强力装备可消耗火种获得特殊效果 |

### 12.2.1 例子：旧文明车载弩炮

| 阶段 | 效果 |
|---|---|
| 残件 | 每场战斗可进行一次不稳定车载射击 |
| 一阶修复 | 车载射击稳定，伤害提高 |
| 二阶修复 | 可和标记/穿甲类效果互动 |
| 火种启动 | 关键战斗中发动更强射击，但消耗火种能量或特殊弹药 |

---

## 12.3 装备成长与强化

### 12.3.1 普通装备成长

普通装备可以多次强化，但本章不写具体曲线。

| 强化方向 | 消耗 | 作用 |
|---|---|---|
| 数值强化 | 金币 + 普通材料 | 提高伤害、护盾、治疗、修理等基础数值 |
| 结构强化 | 金币 + 稀有材料 | 提高机制次数、范围或稳定性 |
| 适配强化 | 职业材料/训练设施 | 让装备更适合某职业 |
| 图纸改造 | 图纸 + 材料 + 金币 | 解锁新功能或分支 |

普通装备不用火种强化。火种主要用于技能词条、记忆相关系统，以及部分旧文明装备启动。

不做：随机词条、传统品质等级、装备耐久、普通装备火种强化、无限数值膨胀。

### 12.3.2 旧文明装备成长

旧文明装备不叫强化，叫修复、启动、适配。

| 方式 | 说明 |
|---|---|
| 修复 | 恢复旧文明装备原有效果 |
| 启动 | 让沉睡或残缺设备重新工作 |
| 适配 | 让角色或商队能够安全使用 |

旧文明装备需要残件、修复图纸/方案、旧文明材料、专门工匠/城市设施。火种只用于部分强力装备启动。

---

## 12.4 装备来源

| 来源 | 产出 |
|---|---|
| 城市商店 | 普通角色装备、基础商队模块、普通图纸 |
| 城镇商店 | 少量人物基础装备，如铁剑、普通护甲 |
| 驿站商店 | 少量商队模块，如装甲板、外设、环境装备 |
| 城市工坊 | 制作、强化、修复 |
| 订单奖励 | 装备、材料、凭证、特殊图纸 |
| 战斗 | 金币、材料、小概率装备 |
| 精英/Boss | 高价值装备、图纸、旧文明残件 |
| 特殊事件 | 破损装备、图纸、旧文明线索 |
| 遗迹探索 | 旧文明装备、修复材料、特殊图纸 |

城镇和驿站不做完整装备中心：城市是主要装备购买、制作、强化点；城镇少量普通人物装备；驿站少量普通商队模块；遗迹产出旧文明装备、稀有材料、特殊图纸。

### 12.4.1 大致掉落/产出概率

| 来源 | 装备/图纸/模块产出概率 |
|---|---:|
| 普通战斗 | 10%–20% 出基础装备、消耗道具或小材料 |
| 精英战斗 | 40%–70% 出装备、图纸、模块或高价值材料 |
| Boss/关键战斗 | 100% 出高价值奖励，包含装备、火种、模块、图纸之一 |
| 普通事件 | 10%–30% 出道具、材料、破损装备 |
| 高风险事件 | 30%–60% 出装备、图纸、特殊道具 |
| 特殊订单 | 50%–100% 按订单类型给装备、图纸或凭证 |
| 遗迹探索 | 中高概率获得材料、图纸、旧文明残件 |

普通战斗不能变成刷装备，但战斗收益也不能太低，否则玩家会无脑避战。精英、Boss、遗迹才是装备和图纸的重要来源。

## 12.5 遗迹探索与订单路线

遗迹探索不是完全独立玩法，而是嵌入跑订单过程。玩家跑订单时，会在路线中遇到遗迹、废墟、旧文明生产基地、被毁城市、地下仓库、旧道路枢纽等探索点。

如果订单时间充裕、补给够、商队状态允许，玩家可以选择进入遗迹探索。探索会消耗天数、补给、药包、零件、商队耐久和订单时间压力，但可能获得金币、材料、图纸、旧文明残件、特殊装备、情报和路线线索。

探索结束后，玩家继续原订单路线。

```text
玩家可以边跑订单，边探索世界。
```

| 遗迹类型 | 可能收益 | 主要风险 |
|---|---|---|
| 被毁城市 | 旧文明残件、金币、图纸、材料 | 敌人多、路线复杂 |
| 废弃生产基地 | 商队模块、制造图纸、零件材料 | 机械敌人、商队耐久损耗 |
| 灰烬污染遗迹 | 灰烬材料、封存货物、旧文明线索 | 灰蚀、污染扩散 |
| 旧道路枢纽 | 路线情报、旧世路引、车载设备 | 伏击、迷路 |
| 地下仓库 | 储存容器、特殊货物、装备残件 | 补给消耗、特殊敌人 |

灰烬污染遗迹只是遗迹类型之一，不是神话终极区。遗迹探索是风险收益选择，不是主线必须完成的副本。

## 12.6 装备制作

普通装备不一定需要图纸。

| 装备类型 | 是否需要图纸 |
|---|---|
| 普通剑、普通弓、普通护甲 | 不需要特殊图纸，城市工坊默认会 |
| 常见商队模块 | 普通版不需要特殊图纸 |
| 复杂装备 | 需要图纸 |
| 职业专属装备 | 需要图纸或记忆解锁 |
| 旧文明装备 | 必须需要修复图纸/修复方案 |
| 火种启动装备 | 必须需要图纸/特殊设施 |

制作要素：金币、普通材料、稀有材料、图纸、旧文明残件、专门工匠/城市设施。

例子：

| 装备 | 需求 |
|---|---|
| 普通弓 | 木材 + 兽筋 + 金币，不需要特殊图纸 |
| 防潮箱 | 防潮布 + 木材 + 金币，普通版不需要特殊图纸 |
| 高级防潮箱 | 防潮箱图纸 + 更好材料 + 金币 |
| 旧文明雾灯 | 旧世透镜 + 雾灯残件 + 修复图纸 + 城市工坊 |

## 12.7 商队牌/车队牌处理

车队牌暂时不单独做大系统。

| 原车队牌内容 | 处理方式 |
|---|---|
| 通行牌、商牌、路引 | 商队凭证槽/挂饰/特殊道具 |
| 主动战术牌 | 尽量少做，控制在 5 张以内 |
| 战斗牌效果 | 转成战术道具或消耗道具 |
| 车辆牌效果 | 转成商队模块特殊效果 |
| 势力证明 | 转成凭证类挂饰或订单道具 |

理由：车队牌如果做太大，会和装备、道具、订单系统重叠。当前只保留轻量接口，不作为第 6.7 的核心大系统。
