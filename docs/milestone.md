# 🏁 里程碑定义文档

> 本文档明确项目各阶段的关键里程碑、验收标准与交付物。

---

## 里程碑总览

```
概念验证 ────→ 原型完成 ────→ Alpha版本 ────→ Beta版本 ────→ 发布候选 ────→ 正式上线
  M0            M1            M2            M3            M4           M5
```

---

## M0: 概念验证完成 (Concept Validation)

**目标日期**：第4周末

### 验收标准
- [ ] 核心玩法概念文档已完成并通过评审
- [ ] 技术可行性评估报告已确认无重大风险
- [ ] 美术概念图已确定整体方向
- [ ] 项目预算与资源计划已获批

### 交付物
- [ ] 《游戏设计概念文档》
- [ ] 《技术可行性评估报告》
- [ ] 《美术概念设计稿》（至少5张）
- [ ] 《项目计划与预算》

### Go/No-Go 决策点
> 在此里程碑需要决定是否正式立项。若核心玩法缺乏吸引力或技术风险过高，应在此阶段终止或调整方向。

---

## M1: 原型完成 / 垂直切片 (Vertical Slice)

**目标日期**：第8周末

### 验收标准
- [ ] 核心玩法可完整体验（至少15分钟游戏时长）
- [ ] 主要角色/单位已完成基础资源制作
- [ ] 至少1个核心关卡/场景可游玩
- [ ] UI框架已搭建，核心界面可用
- [ ] 帧率在目标设备上达到最低标准

### 交付物
- [ ] 可运行的Demo版本
- [ ] 《核心玩法验证报告》
- [ ] 《技术框架说明》
- [ ] 内部试玩反馈汇总

### Go/No-Go 决策点
> 验证核心乐趣是否成立。若试玩反馈不佳，需在此阶段调整玩法或终止项目。

---

## M2: Alpha版本

**目标日期**：第14周末

### 验收标准
- [ ] 所有主要系统功能已实现（可能有Bug）
- [ ] 全部关卡/内容已搭建完毕（可能缺少 polish）
- [ ] 所有美术资源已到位（可能使用临时资源）
- [ ] 游戏可从头到尾通关/体验完整循环
- [ ] 无阻断性Bug（P0级别Bug清零）

### 交付物
- [ ] Alpha版本安装包
- [ ] 《已知问题清单》
- [ ] 《内容完成度检查表》

---

## M3: Beta版本

**目标日期**：第20周末

### 验收标准
- [ ] 所有P0、P1级别Bug已修复
- [ ] 美术资源全部替换为正式资源
- [ ] 音效与音乐全部到位
- [ ] 新手引导与教程完整
- [ ] 性能在各目标设备上达标
- [ ] 本地化内容已完成（如有多语言需求）

### 交付物
- [ ] Beta版本安装包
- [ ] 《测试报告》
- [ ] 《性能分析报告》
- [ ] 外部测试用户反馈

---

## M4: 发布候选版 (Release Candidate)

**目标日期**：第24周末

### 验收标准
- [ ] 所有已知Bug已修复或已评估接受
- [ ] 包体大小符合平台要求
- [ ] 合规审查已通过（年龄分级、隐私政策等）
- [ ] 服务器压力测试通过（如适用）
- [ ] 发布物料已准备完毕（商店页面、宣传素材）

### 交付物
- [ ] RC版本安装包
- [ ] 《发布检查清单》（全部勾选）
- [ ] 商店页面素材
- [ ] 运营活动方案

---

## M5: 正式上线 (Launch)

**目标日期**：第25周

### 验收标准
- [ ] 游戏已在目标平台成功发布
- [ ] 服务器稳定运行（如适用）
- [ ] 客服与社区运营机制已启动
- [ ] 数据监控系统正常运行

### 交付物
- [ ] 正式版游戏
- [ ] 《上线首日数据报告》
- [ ] 《运营计划执行记录》

---

## 风险里程碑 (Risk Checkpoints)

| 检查点 | 时间 | 检查内容 |
|--------|------|---------|
| 技术风险评估 | M0 | 引擎、架构、关键技术的可行性 |
| 美术产能评估 | M1 | 美术风格确定后，评估完整内容产能是否足够 |
| 市场窗口评估 | M2 | 竞品动态、市场时机是否仍然合适 |
| 包体与性能 | M3 | 是否能在限制内优化到位 |
| 合规审查 | M4 | 平台政策、法律合规、年龄分级 |
