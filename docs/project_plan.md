## 2. 详细排期

### 第一阶段：原型开发（第1-6周）

**目标**：做出能跑通 N3.1（灰桥镇 → 第一个驿站）的最小可玩版本。

| 周次 | 任务 | 产出物 |
|------|------|--------|
| W1 | 技术框架搭建：大地图移动、天数系统、基础UI | 可运行的空框架 |
| W1 | 角色数据结构：生命、护甲、牌组、手牌、重伤 | 角色系统原型 |
| W2 | 战斗系统：行动力、抽牌、出牌、防御结算、状态 | 基础战斗可玩 |
| W2 | 三人小队 + 货车系统 | 小队战斗原型 |
| W3 | 三名角色技能实现：护路人、射手、修补师 | 完整技能可用 |
| W3 | 敌人原型：灰烬幼兽、裂背灰烬兽、灰烬腐蚀兽 | 教学敌人可用 |
| W4 | 大地图 + 移动消耗 + 事件触发 | 地图循环可跑 |
| W4 | 补给/零件/金钱 + 城市/驿站基础功能 | 经济循环可用 |
| W5 | N3.1 内容搭建：固定教学路段、灰桥镇、第一场战斗 | N3.1 可玩 |
| W5 | 士气系统 + 扎营/休整 | 士气循环可用 |
| W6 | N3.2 内容搭建：半随机路段、驿站功能、灰烬风暴 | 垂直切片完成 |
| W6 | 内部试玩 + 反馈收集 + 调整 | 试玩报告 |

### 第二阶段：量产阶段（第7-14周）

| 周次 | 任务 |
|------|------|
| W7-8 | N3.3 城市周边经营：工坊、记忆插槽、城市恢复、短途订单 |
| W9-10 | N3.4 开放区域：自由探索、旧遗迹、灰烬边缘、中程路线 |
| W11 | 天气系统：天气段生成、四种天气类别、天气影响 |
| W12 | 外部/内部事件系统 + 事件冷却 |
| W13 | 订单系统完善：多种订单类型、倍率、惩罚 |
| W14 | Boss 战设计 + 火种系统 + 城市重建 |

### 第三阶段：测试优化（第15-18周）

| 周次 | 任务 |
|------|------|
| W15 | 数值全面调优：经济平衡、战斗难度、成长曲线 |
| W16 | 性能优化、包体控制 |
| W17 | 外部测试（小范围） |
| W18 | Bug 修复、最终 polish |

## 3. 技术选型建议

| 项目 | 建议 | 理由 |
|------|------|------|
| **引擎** | Unity 或 Godot | 2D/2.5D 卡牌战斗 + 大地图，团队熟悉度优先 |
| **版本控制** | Git + GitHub | 已在用 |
| **数据配置** | Excel → JSON/ScriptableObject | 现有大量 Excel 母表 |
| **卡牌框架** | 自研或参考开源框架 | 三层防御 + 状态系统较特殊 |

---

## 4. 风险与应对

| 风险 | 影响 | 可能性 | 应对 |
|------|------|--------|------|
| 战斗系统复杂度高 | 高 | 中 | 先实现最小闭环，再扩展状态 |
| Excel 母表转配置工作量大 | 中 | 高 | 早期建立自动导出流程 |
| 三人小队共享行动力的平衡 | 高 | 中 | 持续调优，保留修改空间 |
| 天气/事件系统耦合度高 | 中 | 中 | 模块化设计，事件配置化 |
