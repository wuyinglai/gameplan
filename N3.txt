# 《余烬商队》N3 阶段收口结论

## 一、N3 总定位

N3 阶段的核心是：

> 从开局教学，逐步过渡到正常开放商队循环。

N3 不再只是“新手路线”，而是完成以下几件事：

1. 教会玩家基础战斗；
2. 教会玩家补给、零件、货车、士气；
3. 教会玩家驿站、小镇、村庄、城市的区别；
4. 教会玩家订单、商路、撤退、扎营；
5. 让玩家第一次点亮城市；
6. 让玩家第一次进入开放区域；
7. 让玩家第一次跑正常中程商路。

到 N3.5 结束后，玩家就不再处于新手教程，而是正式进入开放商队玩法。

---

# 二、N3.1 到 N3.5 定稿结构

## N3.1：灰桥镇到第一个驿站

定位：

> 固定教学路段。

内容：

* 从灰桥镇出发；
* 开局固定三人：护路人、荒野射手、修补师；
* 教基础战斗；
* 教补给和零件；
* 教货车耐久；
* 教村庄、小订单、劫匪抢货；
* 最终抵达第一个驿站。

灰桥镇是大型小镇，不是城市。

第一座城市仍然是玩家后续真正点亮的大型据点。

---

## N3.2：第一个驿站到第一座城市

定位：

> 半随机教学路段。

内容：

* 教驿站功能；
* 教灰烬风暴避难；
* 教半随机路线；
* 教远行中继点；
* 最终抵达并点亮第一座城市。

第一座城市可以暂称“大安城”或后续再定名。

---

## N3.3：第一座城市周边经营

定位：

> 城市周边商路重建与经营教学。

内容：

* 开放城市基础功能；
* 开放工坊复产；
* 开放古代记忆插槽；
* 开放城市恢复阶段；
* 跑短途订单；
* 修复周边商路；
* 让玩家理解城市不是靠玩家一个人建设，而是城市自救、其他商队和玩家关键贡献共同推进。

---

## N3.4：第一座城市开放区域

定位：

> 巨大新手教程的最后一段，也是开放世界正式展开的开始。

内容：

* 不规定玩家一定去第二个驿站；
* 玩家可以自由选择继续发展第一座城市；
* 可以跑周边小镇、村庄、驿站；
* 可以探索旧商队残骸、旧文明遗迹、灰烬边缘；
* 可以打听第二座城市传言；
* 可以提前尝试中程路线；
* 通过传言、利润、缺货、其他商队动向，引导玩家自然产生远行欲望。

N3.4 的关键是：

> 不再预判玩家会怎么走，只提供机会和风险。

---

## N3.5：正常中程商路

定位：

> 第一次真正脱离教程的正常中程商路。

基础参数：

| 项目    |          定稿 |
| ----- | ----------: |
| 标准路线  |     约 100 天 |
| 订单期限  |       150 天 |
| 路线结构  |        4 段路 |
| 中继点   |         3 个 |
| 每段距离  |     20–30 天 |
| 安全绕路  | 约 120–125 天 |
| 高风险捷径 |   约 75–85 天 |

中继点可以是：

* 驿站；
* 小镇；
* 村庄；
* 半废弃货站；
* 边境前哨；
* 旧路标节点；
* 灰烬边缘点。

N3.5 不强制玩家去第二座城市。

它只是让玩家第一次完整体验：

> 自己选订单，自己规划路线，自己承担大约 100 天远行的风险和收益。

# 三、地点层级已定

## 灰桥镇

灰桥镇是开局大型小镇。

特点：

* 比普通小镇大；
* 能组织中型商队；
* 补给便宜；
* 初始零件较友好；
* 不开放城市建设、工坊、古代记忆等完整城市系统。

---

## 地点层级

| 地点 | 定位                     |
| -- | ---------------------- |
| 村庄 | 小型聚落，小订单、少量交易、情报       |
| 小镇 | 中型聚落，补给、交易、少量零件、简单休整   |
| 驿站 | 商路中继点，修车、零件、避难、天气等待    |
| 城市 | 大型据点，城市建设、工坊、古代记忆、完整成长 |

# 四、价格规则已定

## 补给价格

| 地点   |      补给价格 |
| ---- | --------: |
| 城市   |   2 金 / 份 |
| 灰桥镇  |   2 金 / 份 |
| 普通小镇 |   3 金 / 份 |
| 驿站   | 3–4 金 / 份 |
| 村庄   | 3–5 金 / 份 |

逻辑：

> 小镇有人长期生活，所以补给比驿站便宜。
> 驿站主要服务过路商队，所以补给偏贵。

---

## 零件价格与库存

| 地点   |        零件价格 |    库存 | 出现概率 |
| ---- | ----------: | ----: | ---: |
| 城市   |     8 金 / 个 |    稳定 | 100% |
| 灰桥镇  |    10 金 / 个 | 5–8 个 | 100% |
| 驿站   | 12–16 金 / 个 | 2–4 个 | 100% |
| 普通小镇 | 18–22 金 / 个 | 0–2 个 |  50% |
| 偏远小镇 | 20–24 金 / 个 | 0–1 个 |  30% |
| 村庄   | 25–30 金 / 个 | 0–1 个 |  10% |

逻辑：

> 小镇适合买补给，驿站适合买零件，城市最稳定，村庄看运气。

# 五、失败、撤退与订单规则已定

## 失败条件

单纯角色全部重伤，不直接失败。

真正严重失败包括：

```text
全部角色重伤
+
货车耐久归零
=
崩溃撤退
```

或者：

```text
士气归零
=
强制撤退
```

---

## 撤退规则

中途撤退不等于订单失败。

订单失败主要看：

```text
订单期限到达
+
目标未送达
=
订单失败
```

撤退分三类：

| 类型     | 触发              |
| ------ | --------------- |
| 主动撤退   | 玩家自己选择          |
| 士气归零撤退 | 士气 = 0          |
| 崩溃撤退   | 全部角色重伤 + 货车耐久归零 |

崩溃撤退代价：

* 订单物品全部丢失；
* 大量金币损失；
* 士气 -4；
* 货车需要高额修理；
* 消耗大量时间；
* 订单不立刻失败，但基本很难继续。

---

# 六、士气与扎营系统已定

## 士气范围

```text
0–10
```

初始建议：

```text
6 / 10
```

士气归零会强制撤退。

---

## 士气预警

| 士气 | 提示       |
| -- | -------- |
| 4  | 低士气预警    |
| 1  | 崩溃边缘强烈预警 |
| 0  | 强制撤退     |

---

## 士气恢复冷却

| 地点        | 恢复 | 条件          |       冷却 |
| --------- | -: | ----------- | -------: |
| 城市 / 大型小镇 | +2 | 士气 ≤ 8      |     30 天 |
| 普通小镇      | +1 | 士气 ≤ 6      |     15 天 |
| 驿站        | +1 | 士气 ≤ 6      |     15 天 |
| 村庄        |  0 | 花钱或完成事件可 +1 | 15 天或一次性 |
| 野外扎营      |  0 | 不恢复士气       |        无 |

冷却按具体地点记录，不按地点类型记录。

---

## 扎营规则

扎营消耗：

* 1 天；
* 双倍补给；
* 货车耐久照常损耗；
* 订单期限继续流逝；
* 扎营事件概率约为正常前进的 50%。

扎营作用：

* 推进角色恢复；
* 允许简单修车；
* 等天气；
* 缓解连续战斗压力；
* 不恢复士气。

---

## 补给归零

补给为 0 后，每过一天：

* 士气 -1；
* 全员生命 -20%；
* 重伤恢复停止。

补给归零不直接失败，但会快速把商队拖垮。

# 七、车队牌系统已加入

战斗中增加：

> 车队牌

每回合行动牌结构：

| 来源   | 数量 |
| ---- | -: |
| 护路人  |  2 |
| 荒野射手 |  2 |
| 修补师  |  2 |
| 车队牌  |  1 |
| 合计   |  7 |

车队牌代表货车和商队整体行动。

基础车队牌包括：

* 货车冲撞；
* 车灯照射；
* 车棚加固；
* 货箱横挡；
* 加固绳索；
* 应急补片。

车队牌让货车不只是被保护对象，而是主动资源。

# 八、N3 当前剩余工作判断

N3 现在不建议继续扩展大系统。

剩余只需要做三件收口工作。

## 1. 做一次总纲合并

把 N3.1 到 N3.5 的结论合并成一份完整 N3 总纲。

目的：

* 避免前后版本冲突；
* 删除旧的“固定去第二个驿站”口径；
* 确认灰桥镇、小镇、士气、撤退、扎营、车队牌都进入正式口径。

---

## 2. 做一次数值冲突检查

重点检查：

* 补给价格；
* 零件价格；
* 士气恢复；
* 撤退惩罚；
* 100 天中程路线；
* 150 天期限；
* 扎营消耗；
* 补给归零惩罚；
* 车队牌消耗货车耐久是否过重。

目的：

> 避免某个系统单独看合理，组合起来过度惩罚玩家。

---

## 3. 留接口，不继续细化

暂时不需要继续把所有事件池写完。

比如：

* 每个小镇具体订单；
* 每条路线具体怪物；
* 每个中继点剧情；
* 每个天气事件；
* 每个传言文本；
* 第二座城市完整结构。

这些可以放到后续阶段。

N3 只需要把“规则骨架”定住。
